# キャッチャー

import { Tab, Tabs } from '@rspress/core/theme';

`Response`のステータスコードがエラーで、ページ内の`Body`が空の場合、Salvoは`Catcher`を使用してこのエラーをキャッチし、ユーザーフレンドリーなエラーページを表示しようとします。

`Catcher::default()`を使用してシステムデフォルトの`Catcher`を返し、それを`Service`に追加することができます。

```rust
use salvo::catcher::Catcher;

Service::new(router).catcher(Catcher::default());
```

デフォルトの`Catcher`は、`XML`、`JSON`、`HTML`、`Text`形式でエラーページを送信することをサポートしています。

このデフォルトの`Catcher`に`hoop`を追加することで、カスタムエラーキャッチプログラムを`Catcher`に追加できます。このエラーキャッチプログラムも依然として`Handler`です。

`hoop`を使用して`Catcher`に複数のカスタムエラーキャッチプログラムを追加できます。カスタムエラーハンドラーは、エラー処理後に`FlowCtrl::skip_next`メソッドを呼び出して後続のエラープログラムをスキップし、早期に戻ることができます。

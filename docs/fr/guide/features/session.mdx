Middleware fournissant la prise en charge de `Session`.

## Qu'est-ce qu'une Session ?

Une Session est un ensemble de données utilisateur stockées côté serveur, généralement utilisées pour suivre l'état de l'utilisateur et conserver des données temporaires. Dans les applications Web, les Sessions permettent de résoudre le problème de l'absence d'état du protocole HTTP, permettant ainsi au serveur d'identifier les requêtes multiples provenant du même utilisateur.

## Utilisation des Sessions dans Salvo

Dans Salvo, le middleware Session fournit une API simple pour lire et écrire des données de session. L'instance de session peut être obtenue via `SessionDepot`, et les données de session peuvent être gérées à l'aide de méthodes telles que `get`, `set`, `remove`, etc.

## Utilisations courantes

- Authentification des utilisateurs et gestion de l'état de connexion
- Stockage temporaire des données de formulaire
- Conservation des préférences utilisateur

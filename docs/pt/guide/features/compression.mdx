# Compressão de Resposta

Middleware para compressão de conteúdo em `Response`.

Oferece suporte a três formatos de compressão: `br`, `gzip`, `deflate`. Permite configurar prioridades e outros parâmetros de cada método de compressão conforme necessário.

## Compressão de dados no protocolo HTTP

1. Nível de formato de arquivo: tipos específicos de arquivo usam algoritmos de compressão especializados
2. Nível de protocolo HTTP: compressão de recursos ponta a ponta através de codificação de conteúdo
3. Nível de conexão: compressão de dados entre nós de conexão HTTP

Para mais informações sobre compressão HTTP, consulte a [documentação da Mozilla Developer Network](https://developer.mozilla.org/pt-BR/docs/Web/HTTP/Guides/Compression).

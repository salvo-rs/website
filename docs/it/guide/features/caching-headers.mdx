# Intestazioni di Cache

Middleware che fornisce supporto per la configurazione delle intestazioni di cache.

Il controllo della cache è una parte fondamentale dell'ottimizzazione delle prestazioni web. Impostando correttamente le intestazioni di cache, è possibile ridurre richieste di rete non necessarie e migliorare le prestazioni dell'applicazione. [Cache-Control](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Reference/Headers/Cache-Control) è un'intestazione di risposta HTTP che specifica le direttive per la strategia di cache del browser, controllando chi può memorizzare in cache la risposta, in quali condizioni e per quanto tempo.

L'implementazione interna include tre gestori (`Handler`): `CachingHeaders`, `Modified` e `ETag`. `CachingHeaders` combina gli ultimi due. In condizioni normali, si utilizza `CachingHeaders`.

- `Modified`: Fornisce convalida della cache basata sull'ultima data di modifica della risorsa.
- `ETag`: Utilizza un tag di entità (Entity Tag) per un meccanismo di convalida delle risorse più preciso.
- `CachingHeaders`: Combina i due meccanismi sopra descritti, fornendo un supporto completo per il controllo della cache.

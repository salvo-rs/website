# JWT Аутентификация

JWT (JSON Web Token) — это открытый стандарт (RFC 7519), предназначенный для безопасной передачи информации между сторонами. Это компактный и URL-безопасный способ представления утверждений в формате JSON-объекта, которые обычно используются для аутентификации и обмена данными.

JWT состоит из трёх частей:
- **Заголовок (Header)** — определяет тип токена и используемый алгоритм подписи
- **Полезная нагрузка (Payload)** — содержит утверждения (claims), такие как ID пользователя, роли, срок действия и т.д.
- **Подпись (Signature)** — используется для проверки целостности сообщения при передаче

Типичный процесс использования JWT в аутентификации:
1. После входа пользователя сервер генерирует JWT-токен
2. Токен возвращается клиенту и сохраняется (обычно в localStorage или cookie)
3. В последующих запросах клиент передаёт токен в заголовке Authorization
4. Сервер проверяет валидность токена и предоставляет доступ

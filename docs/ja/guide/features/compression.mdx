# レスポンス圧縮

`Response` の内容を圧縮処理するミドルウェアです。

3つの圧縮形式に対応しています：`br`、`gzip`、`deflate`。必要に応じて各圧縮方式の優先度などを設定できます。

## HTTP プロトコルにおけるデータ圧縮

1. ファイル形式レベル：特定のファイルタイプに特化した圧縮アルゴリズムを使用
2. HTTP プロトコルレベル：コンテンツエンコーディングによるエンドツーエンドのリソース圧縮転送
3. 接続レベル：HTTP 接続のノード間でのデータ圧縮

HTTP圧縮に関する詳細は [Mozilla デベロッパーネットワークのドキュメント](https://developer.mozilla.org/ja/docs/Web/HTTP/Guides/Compression) をご参照ください。

# Обратный прокси-сервер

Обратный прокси-сервер — это архитектура сервера, которая принимает запросы от клиентов и перенаправляет их на один или несколько внутренних серверов. В отличие от прямого прокси (который действует от имени клиента), обратный прокси работает на стороне сервера.

Основные преимущества обратного прокси:
- Балансировка нагрузки: распределение запросов между несколькими серверами
- Повышение безопасности: скрытие информации о реальном сервере
- Кэширование контента: улучшение производительности
- Перезапись и перенаправление путей: гибкая маршрутизация запросов

Фреймворк Salvo предоставляет промежуточное ПО с функцией обратного прокси.

_**Пример кода**_

```rust
use salvo::prelude::*;
use salvo::proxy::Proxy;

#[tokio::main]
async fn main() {
    tracing_subscriber::fmt().init();
    
    let router = Router::new()
        .push(
            Router::new()
                .path("google/{**rest}")
                .handle(Proxy::<Vec<&str>>::new(vec!["https://www.google.com"])),
        )
        .push(
            Router::new()
                .path("baidu/{**rest}")
                .handle(Proxy::<Vec<&str>>::new(vec!["https://www.baidu.com"])),
        );
    let acceptor = TcpListener::new("127.0.0.1:5800").bind().await; Server::new(acceptor).serve(router).await;
}
```

# Writer

`Writer` используется для записи содержимого в `Response`:

```rust
#[async_trait]
pub trait Writer {
    async fn write(mut self, req: &mut Request, depot: &mut Depot, res: &mut Response);
}
```

В сравнении с Handler:

```rust
#[async_trait]
pub trait Handler: Send + Sync + 'static {
    async fn handle(&self, req: &mut Request, depot: &mut Depot, res: &mut Response, ctrl: &mut FlowCtrl);
}
```

Основные различия между ними:
- Разное назначение: `Writer` представляет запись конкретного содержимого в `Response`, реализуется для определённых типов данных, таких как строки, сообщения об ошибках и т.д. В то время как `Handler` предназначен для обработки всего запроса.
- `Writer` создаётся внутри `Handler` и потребляется при вызове функции `write`, что делает его одноразовым. С другой стороны, `Handler` используется совместно для всех запросов.
- `Writer` может возвращаться как содержимое в `Result` обработчика `Handler`.
- `Writer` не включает параметр `FlowCtrl`, поэтому не может управлять потоком выполнения всего запроса.

`Scribe` реализует `Writer`, но предоставляет меньше возможностей по сравнению с ним:

```rust
pub trait Scribe {
    fn render(self, res: &mut Response);
}
```

Функция рендеринга `Scribe` только записывает данные в `Response` и не может получать информацию из `Request` или `Depot` в процессе выполнения.

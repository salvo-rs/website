# Catcher

import { Tab, Tabs } from '@rspress/core/theme';

Se lo stato della `Response` è un codice di errore e il `Body` nella pagina è vuoto, Salvo tenterà di utilizzare il `Catcher` per intercettare questo errore e visualizzare una pagina di errore amichevole.

È possibile ottenere un `Catcher` predefinito del sistema tramite `Catcher::default()`, quindi aggiungerlo al `Service`.

```rust
use salvo::catcher::Catcher;

Service::new(router).catcher(Catcher::default());
```

Il `Catcher` predefinito supporta l'invio di pagine di errore nei formati `XML`, `JSON`, `HTML` e `Text`.

È possibile aggiungere gestori di errore personalizzati al `Catcher` tramite il metodo `hoop` su questo `Catcher` predefinito. Questi gestori di errore rimangono comunque `Handler`.

È possibile aggiungere più gestori di errore personalizzati al `Catcher` tramite `hoop`. I gestori di errore personalizzati possono chiamare il metodo `FlowCtrl::skip_next` dopo aver gestito l'errore per saltare i successivi gestori e restituire anticipatamente il risultato.

# Аутентификация JWT

JWT (JSON Web Token) — это открытый стандарт (RFC 7519) для безопасной передачи информации между сторонами. Это компактный, URL-безопасный метод представления утверждений в формате JSON-объекта, обычно используемый для аутентификации и обмена информацией.

JWT состоит из трёх частей:
- **Заголовок** — определяет тип токена и используемый алгоритм подписи
- **Полезная нагрузка** — содержит утверждения, такие как ID пользователя, роли, время истечения срока действия и т.д.
- **Подпись** — используется для проверки неизменности сообщения при передаче

Типичное использование JWT в процессе аутентификации:
1. После входа пользователя сервер генерирует JWT-токен
2. Токен возвращается клиенту и сохраняется (обычно в localStorage или cookies)
3. В последующих запросах клиент включает этот токен в заголовок Authorization
4. Сервер проверяет подлинность токена и предоставляет доступ

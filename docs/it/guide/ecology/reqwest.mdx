# Reqwest: Libreria client HTTP per Rust

[Reqwest](https://docs.rs/reqwest/latest/reqwest/) è una libreria client HTTP di alto livello che semplifica la gestione delle richieste HTTP, offrendo numerose funzionalità comuni:

- Supporto per API asincrone e sincrone (bloccanti)
- Gestione di diversi tipi di corpo delle richieste: testo semplice, JSON, form codificati URL, form multipart
- Politica di reindirizzamento personalizzabile
- Supporto per proxy HTTP
- Crittografia TLS abilitata di default
- Gestione dei cookie

## Utilizzo di base

### Effettuare una richiesta GET

Per una singola richiesta, è possibile utilizzare il metodo rapido `get`:

```rust
let body = reqwest::get("https://www.rust-lang.org")
    .await?
    .text()
    .await?;

println!("body = {body:?}");
```

> Nota: se prevedi di effettuare più richieste, è preferibile creare un `Client` e riutilizzarlo per sfruttare i vantaggi del connection pooling.

```rust
let client = reqwest::Client::new();
let res = client.get("https://www.rust-lang.org")
    .send()
    .await?;
```

### Effettuare una richiesta POST

Il metodo `body()` permette di impostare il corpo della richiesta:

```rust
let client = reqwest::Client::new();
let res = client.post("http://httpbin.org/post")
    .body("il contenuto esatto da inviare")
    .send()
    .await?;
```

### Dati di form

Inviare dati di form è un'esigenza comune. È possibile utilizzare qualsiasi tipo serializzabile in un form:

```rust
// Questo invierà una richiesta POST con corpo `foo=bar&baz=quux`
let params = [("foo", "bar"), ("baz", "quux")];
let client = reqwest::Client::new();
let res = client.post("http://httpbin.org/post")
    .form(&params)
    .send()
    .await?;
```

### Dati JSON

Il metodo `json` permette di inviare facilmente dati JSON (richiede la feature `json`):

```rust
// Questo invierà una richiesta POST con corpo `{"lang":"rust","body":"json"}`
let mut map = HashMap::new();
map.insert("lang", "rust");
map.insert("body", "json");

let client = reqwest::Client::new();
let res = client.post("http://httpbin.org/post")
    .json(&map)
    .send()
    .await?;
```

### Gestione della risposta

La risposta può essere gestita in diversi modi:

```rust
let res = client.get("https://www.rust-lang.org").send().await?;

// Ottenere il codice di stato
let status = res.status();

// Ottenere le intestazioni della risposta
let content_type = res.headers().get("content-type").unwrap();

// Leggere il corpo della risposta come testo
let body = res.text().await?;

// Oppure parsarlo come JSON
let json: serde_json::Value = res.json().await?;
```

## Funzionalità avanzate

### Politica di reindirizzamento

Di default, il client gestisce automaticamente i reindirizzamenti HTTP, seguendo fino a 10 salti. È possibile personalizzare questo comportamento utilizzando `ClientBuilder`:

```rust
let custom_client = reqwest::Client::builder()
    .redirect(reqwest::redirect::Policy::none()) // Disabilita i reindirizzamenti
    .build()?;
```

### Supporto per i cookie

È possibile abilitare la memorizzazione e l'invio automatico dei cookie di sessione tramite `ClientBuilder`:

```rust
let client = reqwest::Client::builder()
    .cookie_store(true)
    .build()?;
```

### Configurazione del proxy

Il proxy di sistema è abilitato di default, cercando le impostazioni del proxy HTTP o HTTPS nelle variabili d'ambiente:

- `HTTP_PROXY` o `http_proxy`: proxy per connessioni HTTP
- `HTTPS_PROXY` o `https_proxy`: proxy per connessioni HTTPS
- `ALL_PROXY` o `all_proxy`: proxy per entrambi i tipi di connessione

È anche possibile impostare un proxy esplicitamente nel codice:

```rust
let proxy = reqwest::Proxy::http("https://secure.example")?;
let client = reqwest::Client::builder()
    .proxy(proxy)
    .build()?;

// Oppure disabilitare i proxy
let client = reqwest::Client::builder()
    .no_proxy()
    .build()?;
```

### Configurazione TLS

Il client utilizza TLS di default per connettersi a destinazioni HTTPS:

```rust
// Aggiungere certificati server aggiuntivi
let cert = reqwest::Certificate::from_pem(&cert_bytes)?;
let client = reqwest::Client::builder()
    .add_root_certificate(cert)
    .build()?;

// Configurare certificati client
let identity = reqwest::Identity::from_pkcs12_der(&pkcs12_der, "password")?;
let client = reqwest::Client::builder()
    .identity(identity)
    .build()?;
```

### Impostazioni di timeout

È possibile configurare i timeout per le richieste:

```rust
let client = reqwest::Client::builder()
    .timeout(std::time::Duration::from_secs(10))
    .build()?;
```

## Funzionalità opzionali

Reqwest offre diverse funzionalità opzionali che possono essere abilitate o disabilitate tramite feature di Cargo:

- `http2` (abilitata di default): supporto per HTTP/2
- `default-tls` (abilitata di default): supporto TLS per HTTPS
- `rustls-tls`: fornisce funzionalità TLS utilizzando rustls
- `blocking`: fornisce API client sincrone (bloccanti)
- `json`: fornisce serializzazione e deserializzazione JSON
- `multipart`: fornisce funzionalità per form multipart
- `cookies`: fornisce supporto per le sessioni con cookie
- `gzip`, `brotli`, `deflate`, `zstd`: supporto per la decompressione del corpo della risposta
- `socks`: supporto per proxy SOCKS5

## API sincrone (bloccanti)

Quando non è necessaria l'operatività asincrona, è possibile utilizzare l'API sincrona (richiede la feature `blocking`):

```rust
let body = reqwest::blocking::get("https://www.rust-lang.org")?.text()?;

let client = reqwest::blocking::Client::new();
let res = client.post("http://httpbin.org/post")
    .json(&map)
    .send()?;
```

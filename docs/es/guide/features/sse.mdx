## ¿Qué es SSE?

Server-Sent Events (SSE) es una tecnología web que permite al servidor enviar datos al cliente de forma automática. A diferencia de WebSocket, SSE es una comunicación unidireccional: solo transmite datos del servidor al cliente, lo que lo hace ideal para escenarios donde el servidor necesita enviar datos en tiempo real sin que el cliente tenga que enviar información con frecuencia. SSE se basa en el protocolo HTTP, es sencillo de implementar y cuenta con un mecanismo de reconexión automática en caso de interrupción.

## Integración con modelos de lenguaje grandes

Al integrar IA mediante SDKs como OpenAI u otros SDKs de modelos de lenguaje grandes, la devolución de resultados en flujo continuo (Stream) es un escenario de uso común. SSE ofrece una solución perfecta, permitiendo enviar contenido generado por la IA al frontend en tiempo real, mejorando así la experiencia del usuario.

Por ejemplo, al utilizar la API de Chat Completion de OpenAI con respuesta en flujo continuo, se puede aprovechar la funcionalidad SSE de Salvo para enviar progresivamente el contenido generado por la IA al cliente, evitando la sensación de demora que produce esperar la respuesta completa.

# Catcher

import { Tab, Tabs } from '@rspress/core/theme';

Se o código de status da `Response` for um erro e o `Body` na página estiver vazio, o Salvo tentará usar o `Catcher` para capturar esse erro e exibir uma página de erro amigável.

Você pode usar `Catcher::default()` para retornar um `Catcher` padrão do sistema e, em seguida, adicioná-lo ao `Service`.

```rust
use salvo::catcher::Catcher;

Service::new(router).catcher(Catcher::default());
```

O `Catcher` padrão suporta o envio de páginas de erro nos formatos `XML`, `JSON`, `HTML` e `Text`.

Você pode adicionar manipuladores de erro personalizados ao `Catcher` adicionando um `hoop` a esse `Catcher` padrão. Esse manipulador de erro ainda é um `Handler`.

Você pode adicionar vários manipuladores de erro personalizados ao `Catcher` usando `hoop`. Um manipulador de erro personalizado pode chamar o método `FlowCtrl::skip_next` após processar o erro para pular os manipuladores subsequentes e retornar antecipadamente.

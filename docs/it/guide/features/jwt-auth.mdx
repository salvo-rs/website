# Autenticazione JWT

JWT (JSON Web Token) è uno standard aperto (RFC 7519) per trasmettere informazioni in modo sicuro tra le parti. È un metodo compatto e URL-safe per rappresentare dichiarazioni sotto forma di oggetto JSON, utilizzate tipicamente per autenticazione e scambio di informazioni.

Un JWT è composto da tre parti:
- **Header** - Specifica il tipo di token e l'algoritmo di firma utilizzato
- **Payload** - Contiene le dichiarazioni (claims) come ID utente, ruoli, data di scadenza, ecc.
- **Signature** - Utilizzata per verificare che il messaggio non sia stato alterato durante la trasmissione

Flusso tipico di utilizzo JWT nell'autenticazione:
1. Dopo il login, il server genera un token JWT
2. Il token viene restituito al client e memorizzato (solitamente in localStorage o cookie)
3. Nelle richieste successive, il client include il token nell'header Authorization
4. Il server verifica la validità del token e autorizza l'accesso

Fornisce un middleware per l'autenticazione JWT.

# Использование шаблонизаторов

import { Tab, Tabs } from 'rspress/theme';
import MainCode from '../../../../codes_md/template-askama/src/main.mdx';
import CargoCode from '../../../../codes_md/template-askama/Cargo.mdx';
import TemplateCode from '../../../../codes_md/template-askama/templates/hello.mdx';

Salvo не включает встроенные шаблонизаторы, поскольку предпочтения в стиле шаблонизаторов у всех разные.

По своей сути шаблонизатор — это: данные + шаблон = строка.

Поэтому можно использовать любой шаблонизатор, который способен рендерить итоговую строку.

Например, поддержка `askama`:

<Tabs>
  <Tab label="main.rs">
    <MainCode/>
  </Tab>
  <Tab label="Cargo.toml">
    <CargoCode/>
  </Tab>
  <Tab label="template/hello.toml">
    <TemplateCode/>
  </Tab>
</Tabs>

**Примечание**: Для не слишком сложных проектов мы рекомендуем использовать подход с разделением фронтенда и бэкенда, применяя более гибкие и развитые UI-фреймворки (такие как React, Vue, Svelte и др.) для построения фронтенда, а Salvo — в качестве бэкенд-API. Такой подход повышает эффективность разработки, четче разделяет обязанности между фронтендом и бэкендом и больше соответствует современным тенденциям разработки веб-приложений.

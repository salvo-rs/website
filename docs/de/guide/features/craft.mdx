# Craft-Funktionalität

Craft ermöglicht Entwicklern, durch einfache Annotationen automatisch Handler-Funktionen und Endpunkte zu generieren und nahtlos mit der OpenAPI-Dokumentationserstellung zu integrieren.

## Anwendungsfälle

Die Craft-Funktionalität ist in folgenden Szenarien besonders nützlich:

- Wenn Sie aus Strukturmethoden schnell Routing-Handler-Funktionen erstellen müssen
- Wenn Sie manuell geschriebenen Boilerplate-Code für Parameter-Extraktion und Fehlerbehandlung reduzieren möchten
- Wenn Sie für Ihre API automatisch OpenAPI-Dokumentation generieren müssen
- Wenn Sie Geschäftslogik vom Web-Framework entkoppeln möchten

## Grundlegende Verwendung

Um die Craft-Funktionalität zu nutzen, müssen folgende Module importiert werden:

```rust
use salvo::oapi::extract::*;
use salvo::prelude::*;
```

### Service-Struktur erstellen

Durch Annotation Ihres impl-Blocks mit dem `#[craft]`-Makro können Strukturmethoden in Handler-Funktionen oder Endpunkte umgewandelt werden.

```rust
#[derive(Clone)]
pub struct Opts {
    state: i64,
}

#[craft]
impl Opts {
    // Konstruktor
    fn new(state: i64) -> Self {
        Self { state }
    }
    
    // Weitere Methoden...
}
```

### Handler-Funktion erstellen

Verwenden Sie `#[craft(handler)]`, um eine Methode in eine Handler-Funktion umzuwandeln:

```rust
#[craft(handler)]
fn add1(&self, left: QueryParam<i64>, right: QueryParam<i64>) -> String {
    (self.state + *left + *right).to_string()
}
```

Diese Methode wird zu einer Handler-Funktion, die:
- Automatisch die Werte `left` und `right` aus Query-Parametern extrahiert
- Auf den `state`-Status in der Struktur zugreift
- Das Berechnungsergebnis als String-Antwort zurückgibt

### Endpunkt erstellen

Verwenden Sie `#[craft(endpoint)]`, um eine Methode in einen Endpunkt umzuwandeln:

```rust
#[craft(endpoint)]
pub(crate) fn add2(
    self: ::std::sync::Arc<Self>,
    left: QueryParam<i64>,
    right: QueryParam<i64>,
) -> String {
    (self.state + *left + *right).to_string()
}
```

Endpunkte können `Arc` nutzen, um Status zu teilen, was besonders bei der Verarbeitung gleichzeitiger Anfragen nützlich ist.

### Statischer Endpunkt

Sie können auch statische Endpunkte erstellen, die nicht vom Instanzstatus abhängen:

```rust
#[craft(endpoint(responses((status_code = 400, description = "Wrong request parameters."))))]
pub fn add3(left: QueryParam<i64>, right: QueryParam<i64>) -> String {
    (*left + *right).to_string()
}
```

In diesem Beispiel wurde auch eine Beschreibung für benutzerdefinierte Fehlerantworten hinzugefügt, die sich in der generierten OpenAPI-Dokumentation widerspiegelt.

## Parameter-Extraktoren

Das `oapi::extract`-Modul von Salvo bietet verschiedene Parameter-Extraktoren, die häufigsten sind:

- `QueryParam<T>`: Extrahiert Parameter aus der Query-Zeichenfolge
- `PathParam<T>`: Extrahiert Parameter aus dem URL-Pfad
- `FormData<T>`: Extrahiert Parameter aus Formulardaten
- `JsonBody<T>`: Extrahiert Parameter aus dem JSON-Anfragekörper

Diese Extraktoren führen automatisch Parameter-Parsing und Typkonvertierung durch und vereinfachen so die Erstellung von Handler-Funktionen erheblich.

## Integration mit OpenAPI

Die Craft-Funktionalität kann automatisch API-Dokumentation generieren, die der OpenAPI-Spezifikation entspricht. Im Beispiel:

```rust
let router = Router::new()
    .push(Router::with_path("add1").get(opts.add1()))
    .push(Router::with_path("add2").get(opts.add2()))
    .push(Router::with_path("add3").get(Opts::add3()));

// OpenAPI-Dokumentation generieren
let doc = OpenApi::new("Example API", "0.0.1").merge_router(&router);

// OpenAPI-Dokumentation und Swagger-UI-Routing hinzufügen
let router = router
    .push(doc.into_router("/api-doc/openapi.json"))
    .push(SwaggerUi::new("/api-doc/openapi.json").into_router("swagger-ui"));
```

Nach dieser Konfiguration wird die API-Dokumentation unter dem Endpunkt `/api-doc/openapi.json` bereitgestellt und die Swagger-UI unter dem Pfad `/swagger-ui` verfügbar sein.

## Vollständiges Beispiel

Hier ist ein vollständiges Beispiel, das zeigt, wie die Craft-Funktionalität verwendet wird, um drei verschiedene Arten von Endpunkten zu erstellen:

```rust
use salvo::oapi::extract::*;
use salvo::prelude::*;
use std::sync::Arc;

#[derive(Clone)]
pub struct Opts {
    state: i64,
}

#[craft]
impl Opts {
    fn new(state: i64) -> Self {
        Self { state }
    }

    #[craft(handler)]
    fn add1(&self, left: QueryParam<i64>, right: QueryParam<i64>) -> String {
        (self.state + *left + *right).to_string()
    }

    #[craft(endpoint)]
    pub(crate) fn add2(
        self: ::std::sync::Arc<Self>,
        left: QueryParam<i64>,
        right: QueryParam<i64>,
    ) -> String {
        (self.state + *left + *right).to_string()
    }

    #[craft(endpoint(responses((status_code = 400, description = "Wrong request parameters."))))]
    pub fn add3(left: QueryParam<i64>, right: QueryParam<i64>) -> String {
        (*left + *right).to_string()
    }
}

#[tokio::main]
async fn main() {
    // Gemeinsamen Status mit Anfangswert 1 erstellen
    let opts = Arc::new(Opts::new(1));

    // Routing für drei Endpunkte konfigurieren
    let router = Router::new()
        .push(Router::with_path("add1").get(opts.add1()))
        .push(Router::with_path("add2").get(opts.add2()))
        .push(Router::with_path("add3").get(Opts::add3()));

    // OpenAPI-Dokumentation generieren
    let doc = OpenApi::new("Example API", "0.0.1").merge_router(&router);
    
    // OpenAPI-Dokumentation und Swagger-UI-Routing hinzufügen
    let router = router
        .push(doc.into_router("/api-doc/openapi.json"))
        .push(SwaggerUi::new("/api-doc/openapi.json").into_router("swagger-ui"));

    // Server auf localhost:5800 starten
    let acceptor = TcpListener::new("127.0.0.1:5800").bind().await;
    Server::new(acceptor).serve(router).await;
}
```

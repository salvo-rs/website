# 壓縮回應

對 `Response` 內容進行壓縮處理的中介軟體。

提供三種壓縮格式支援：`br`、`gzip`、`deflate`。可根據需求配置各壓縮方式的優先順序等參數。

## HTTP 協定中的資料壓縮

1. 檔案格式層面：特定檔案類型使用專用壓縮演算法
2. HTTP 協定層面：透過內容編碼實現端到端的資源壓縮傳輸
3. 連線層面：在 HTTP 連線的節點之間進行資料壓縮

更多關於 HTTP 壓縮的資訊可參考 [Mozilla 開發者網路文件](https://developer.mozilla.org/zh-TW/docs/Web/HTTP/Guides/Compression)。

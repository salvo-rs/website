# Compresión de Respuesta

Middleware para el procesamiento de compresión del contenido de `Response`.

Ofrece soporte para tres formatos de compresión: `br`, `gzip`, `deflate`. Se puede configurar la prioridad de cada método de compresión según las necesidades.

## Compresión de Datos en el Protocolo HTTP

1. Nivel de formato de archivo: tipos de archivo específicos utilizan algoritmos de compresión especializados
2. Nivel de protocolo HTTP: implementa transmisión de recursos comprimidos de extremo a extremo mediante codificación de contenido
3. Nivel de conexión: compresión de datos entre nodos de conexión HTTP

Para más información sobre compresión HTTP, consulta la [documentación de Mozilla Developer Network](https://developer.mozilla.org/es/docs/Web/HTTP/Guides/Compression).

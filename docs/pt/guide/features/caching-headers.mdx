# Cabeçalhos de Cache

Um middleware que fornece suporte para configurar cabeçalhos de cache.

O controle de cache é uma parte crucial da otimização de desempenho web. Ao configurar corretamente os cabeçalhos de cache, é possível reduzir solicitações de rede desnecessárias, melhorando assim o desempenho da aplicação. [Cache-Control](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Cache-Control) é um cabeçalho de resposta HTTP usado para especificar políticas de cache do navegador. Ele controla quem pode armazenar respostas em cache, em quais condições e por quanto tempo.

Internamente, inclui implementações de três `Handler`s: `CachingHeaders`, `Modified` e `ETag`. `CachingHeaders` é uma combinação dos dois últimos. Em circunstâncias normais, utiliza-se `CachingHeaders`.

- `Modified`: Fornece validação de cache baseada no último horário de modificação do recurso.
- `ETag`: Usa Entity Tags para oferecer um mecanismo de validação de recursos mais preciso.
- `CachingHeaders`: Combina os dois mecanismos acima para fornecer suporte abrangente ao controle de cache.

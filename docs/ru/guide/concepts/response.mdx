# Response

В `Handler` [`Response`](https://docs.rs/salvo_core/latest/salvo_core/http/response/struct.Response.html) передаётся в качестве параметра:
* Структура `Response` инкапсулирует все компоненты HTTP-ответа, предоставляя полный API для построения и управления HTTP-ответами.
* Поддерживает цепочечный стиль вызовов (например, `res.status_code(200).body("Hello")`), что удобно для плавного построения ответа.
* Ключевые возможности включают:
    * Установку кода состояния и заголовков.
    * Управление телом ответа (поддерживаются строки, байты, файлы и потоковые данные).
    * Управление Cookie.
    * Различные способы рендеринга контента.
* Структура использует шаблон изменяемой ссылки, возвращая ссылку на себя через `&mut self`, что позволяет обработчикам удобно строить и настраивать HTTP-ответы для удовлетворения различных потребностей веб-сервисов.

```rust
#[handler]
async fn hello_world(res: &mut Response) {
    res.render("Hello world!");
}
```

После того как сервер получает запрос от клиента, любой соответствующий `Handler` или промежуточное ПО может записывать данные в `Response`. В некоторых случаях, например, когда промежуточное ПО хочет предотвратить выполнение последующих промежуточных обработчиков и `Handler`, вы можете использовать `FlowCtrl`:

```rust
#[handler]
async fn hello_world(res: &mut Response, ctrl: &mut FlowCtrl) {
    ctrl.skip_rest();
    res.render("Hello world!");
}
```

## Запись контента

Запись данных в `Response` очень проста:

- Запись простых текстовых данных

    ```rust
    res.render("Hello world!");
    ```

- Запись сериализованных JSON-данных

    ```rust
    use serde::Serialize;
    use salvo::prelude::Json;

    #[derive(Serialize, Debug)]
    struct User {
        name: String,
    }
    let user = User{name: "jobs".to_string()};
    res.render(Json(user));
    ```

:::warning
При многократном вызове метода `render` для записи JSON-данных эти данные не объединяются в один JSON-объект, а последовательно объединяются как независимые текстовые фрагменты, что может привести к недопустимому формату JSON. Если необходимо вернуть несколько данных, их следует объединить в один объект перед сериализацией или обработать логику самостоятельно.
:::

- Запись HTML

    ```rust
    res.render(Text::Html("<html><body>hello</body></html>"));
    ```

## Запись HTTP-ошибок

- Использование `render` позволяет записать в `Response` детальную информацию об ошибке.

    ```rust
    use salvo::http::errors::*;
    res.render(StatusError::internal_server_error().brief("error when serialize object to json"))
    ```

- Если вам не нужна пользовательская информация об ошибке, вы можете напрямую вызвать `set_http_code`.

    ```rust
    use salvo::http::StatusCode;
    res.status_code(StatusCode::BAD_REQUEST);
    ```

## Перенаправление на другой URL
- Использование метода `render` позволяет записать в `Response` ответ с перенаправлением на новый URL. При вызове метода `Redirect::found` устанавливается HTTP-код состояния 302 (Found), что указывает на временное перенаправление.
    ```rust
    use salvo::prelude::*;

    #[handler]
    async fn redirect(res: &mut Response) {
        res.render(Redirect::found("https://salvo.rs/"));
    }
    ```

## ResBody

Тип Body, возвращаемый Response, — это `ResBody`, который является перечислением. В случае ошибки он устанавливается в `ResBody::Error`, содержащий информацию об ошибке для последующей обработки. `StatusError` фактически не реализует `Writer`, что позволяет вам настраивать собственный способ отображения в `Catcher`.

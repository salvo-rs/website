# Writer

`Writer` используется для записи содержимого в `Response`:

```rust
#[async_trait]
pub trait Writer {
    async fn write(mut self, req: &mut Request, depot: &mut Depot, res: &mut Response);
}
```

В сравнении с `Handler`:

```rust
#[async_trait]
pub trait Handler: Send + Sync + 'static {
    async fn handle(&self, req: &mut Request, depot: &mut Depot, res: &mut Response, ctrl: &mut FlowCtrl);
}
```

Основные различия между ними:
- Назначение разное: `Writer` предназначен для записи конкретного содержимого в `Response` и реализуется для конкретных типов данных, таких как строки, сообщения об ошибках и т.д. В то время как `Handler` используется для обработки всего запроса.
- `Writer` создается внутри `Handler` и потребляется при вызове функции `write`, что делает его одноразовым. В отличие от этого, `Handler` является общим для всех запросов.
- `Writer` может быть возвращен как содержимое `Result` в `Handler`.
- В `Writer` отсутствует параметр `FlowCtrl`, поэтому он не может управлять процессом выполнения запроса.

`Scribe` реализует `Writer`, но предоставляет меньше возможностей:

```rust
pub trait Scribe {
    fn render(self, res: &mut Response);
}
```

Функция рендеринга `Scribe` только записывает данные в `Response`, и этот процесс не позволяет получать информацию из `Request` или `Depot`.

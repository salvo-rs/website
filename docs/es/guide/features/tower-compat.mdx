# Compatibilidad con Middleware Tower

Salvo proporciona soporte de compatibilidad para el ecosistema Tower a través de la funcionalidad `tower-compat`. Para las APIs específicas, por favor [consulta la documentación](https://docs.rs/salvo_extra/latest/salvo_extra/tower_compat/index.html).

## Conceptos Fundamentales de Tower

Tower es una biblioteca de abstracción de servicios basada principalmente en dos rasgos (traits) centrales:

### Rasgo Service

```rust
pub trait Service<Request> {
    type Response;
    type Error;
    type Future: Future<Output = Result<Self::Response, Self::Error>>;

    fn poll_ready(&mut self, cx: &mut Context<'_>) -> Poll<Result<(), Self::Error>>;
    fn call(&mut self, req: Request) -> Self::Future;
}
```

El rasgo `Service` es responsable de manejar peticiones y devolver respuestas, similar a los manejadores (handlers) de Salvo.

### Rasgo Layer

```rust
pub trait Layer<S> {
    type Service;
    fn layer(&self, inner: S) -> Self::Service;
}
```

El rasgo `Layer` se utiliza para envolver servicios y añadir funcionalidad adicional, similar al middleware de Salvo.

## Soporte de Compatibilidad de Salvo

Salvo proporciona dos rasgos clave para lograr compatibilidad con el ecosistema Tower:

- **TowerLayerCompat**: Convierte el `Layer` de Tower en un `Handler` de Salvo, permitiendo su uso como un Hoop (middleware).
- **TowerServiceCompat**: Convierte el `Service` de Tower en un `Handler` de Salvo.

Ejemplo de uso:
```rust
// Usa RateLimitLayer de Tower y lo convierte a un formato compatible con Salvo mediante el método compat()
let limit = RateLimitLayer::new(5, Duration::from_secs(30)).compat();
let router = Router::new().hoop(limit).get(hello);
```

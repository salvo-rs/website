const DB_URL: &str = "postgres://benchmarkdbuser:benchmarkdbpass@tfb-database/hello_world";
type PgPool = Pool<ConnectionManager<PgConnection>>;

static DB_POOL: OnceCell<PgPool> = OnceCell::new();

fn connect() -> Result<PooledConnection<ConnectionManager<PgConnection>>, PoolError> {
    DB_POOL.get().unwrap().get()
}
fn build_pool(database_url: &str, size: u32) -> Result<PgPool, PoolError> {
    let manager = ConnectionManager::<PgConnection>::new(database_url);
    diesel::r2d2::Pool::builder()
        .max_size(size)
        .min_idle(Some(size))
        .test_on_check_out(false)
        .idle_timeout(None)
        .max_lifetime(None)
        .build(manager)
}

fn main() {
    DB_POOL
        .set(build_pool(&DB_URL, 10).expect(&format!("Error connecting to {}", &DB_URL)))
        .ok();
}

## [SeaORM](https://www.sea-ql.org/SeaORM/)

SeaORM es un ORM asíncrono y dinámico que proporciona un sólido soporte para bases de datos relacionales, incluyendo relaciones entre entidades, un sistema de migraciones y un completo constructor de consultas con seguridad de tipos. Es ideal para proyectos de mediana a gran escala que requieren un ORM con todas las funcionalidades.

## [Tokio ORM (Toasty)](https://github.com/tokio-rs/toasty)

Toasty es un ORM desarrollado por el equipo de Tokio, actualmente en desarrollo activo. Se centra en proporcionar una solución ORM estrechamente integrada para el ecosistema Tokio. Puede ser adecuado para proyectos que dependen de Tokio y están abiertos a adoptar tecnologías emergentes.

## [SurrealDB Rust SDK](https://surrealdb.com/docs/sdk/rust)

El SDK de Rust para SurrealDB proporciona conectividad con esta base de datos multimodelo, lo que la hace adecuada para aplicaciones que necesitan manejar datos de grafos, datos de documentos y datos relacionales. Es una excelente opción para proyectos que requieren modelos de datos flexibles.

# Compatibilità con i Middleware Tower

import { Tab, Tabs } from '@rspress/core/theme';

Salvo offre supporto alla compatibilità con l'ecosistema Tower attraverso la funzionalità `tower-compat`. Per le API specifiche, consultare la [documentazione](https://docs.rs/salvo_extra/latest/salvo_extra/tower_compat/index.html).

## Concetti Fondamentali di Tower

Tower è una libreria di astrazione dei servizi, basata principalmente su due tratti fondamentali:

### Il tratto Service

```rust
pub trait Service<Request> {
    type Response;
    type Error;
    type Future: Future<Output = Result<Self::Response, Self::Error>>;

    fn poll_ready(&mut self, cx: &mut Context<'_>) -> Poll<Result<(), Self::Error>>;
    fn call(&mut self, req: Request) -> Self::Future;
}
```

Il `Service` è responsabile di elaborare le richieste e restituire le risposte, simile a un gestore (handler) in Salvo.

### Il tratto Layer

```rust
pub trait Layer<S> {
    type Service;
    fn layer(&self, inner: S) -> Self::Service;
}
```

Il `Layer` viene utilizzato per avvolgere un servizio e aggiungere funzionalità extra, simile a un middleware in Salvo.

## Supporto alla Compatibilità di Salvo

Salvo fornisce l'implementazione di due tratti chiave per garantire la compatibilità con l'ecosistema Tower:

- **TowerLayerCompat**: Converte il `Layer` di Tower in un `Handler` di Salvo, permettendone l'uso come Hoop (middleware).
- **TowerServiceCompat**: Converte il `Service` di Tower in un `Handler` di Salvo.

Esempio di utilizzo:
```rust
// Utilizza il RateLimitLayer di Tower e lo converte in formato compatibile con Salvo tramite il metodo compat()
let limit = RateLimitLayer::new(5, Duration::from_secs(30)).compat();
let router = Router::new().hoop(limit).get(hello);
```

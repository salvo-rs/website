# Контроль трафика

Промежуточное ПО, предоставляющее функции контроля трафика.

## Основные возможности

* `RateIssuer` предоставляет абстракцию для назначения ключа, идентифицирующего посетителя. `RemoteIpIssuer` является одной из его реализаций, которая может определять посетителя на основе IP-адреса запроса. Ключ не обязательно должен быть строкового типа; любой тип, удовлетворяющий ограничениям `Hash + Eq + Send + Sync + 'static`, может использоваться в качестве ключа.

* `RateGuard` предоставляет абстракцию для алгоритма контроля трафика. По умолчанию реализованы два метода: фиксированное окно (`FixedGuard`) и скользящее окно (`SlidingGuard`).

* `RateStore` предоставляет операции доступа к данным. `MokaStore` — это встроенная реализация кэша в памяти на основе `moka`. Вы также можете определить собственную реализацию.

* `RateLimiter` — это структура, реализующая `Handler`, которая также содержит поле `skipper`, позволяющее указать, какие запросы следует пропустить без кэширования. По умолчанию используется `none_skipper`, который не пропускает никакие запросы.

* `QuotaGetter` предоставляет абстракцию для получения квоты. На основе ключа (`Key`) посетителя можно получить объект квоты, что означает возможность хранения информации о квотах пользователей (например, в базе данных) и их динамического изменения и получения.

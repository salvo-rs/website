# Заголовки кэширования

Промежуточное ПО, обеспечивающее поддержку настройки заголовков кэширования.

Управление кэшированием является важной частью оптимизации веб-производительности. Правильная настройка заголовков кэширования позволяет уменьшить количество ненужных сетевых запросов и повысить производительность приложения. [Cache-Control](https://developer.mozilla.org/ru/docs/Web/HTTP/Headers/Cache-Control) — это директива в заголовке HTTP-ответа, используемая для указания стратегии кэширования браузера. Она контролирует, кто, при каких условиях и как долго может кэшировать ответ.

Фактически, внутренняя реализация включает три обработчика: `CachingHeaders`, `Modified` и `ETag`. `CachingHeaders` является комбинацией двух последних. В обычных ситуациях рекомендуется использовать `CachingHeaders`.

- `Modified`: Обеспечивает проверку кэша на основе времени последнего изменения ресурса.
- `ETag`: Использует тег сущности (Entity Tag) для более точного механизма проверки ресурсов.
- `CachingHeaders`: Объединяет два вышеуказанных механизма, предоставляя полную поддержку управления кэшированием.

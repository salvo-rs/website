# WebTransport

WebTransport — это сетевой протокол передачи данных на основе HTTP/3, обеспечивающий двустороннюю связь между клиентом и сервером, сочетая низкую задержку, высокую пропускную способность и безопасность.

## Области применения

WebTransport особенно подходит для следующих сценариев:

- **Приложения реального времени**: онлайн-игры, инструменты совместной работы, видеоконференции и другие приложения, требующие низкой задержки связи
- **Передача больших файлов**: поддержка передачи данных с высокой пропускной способностью, подходит для медиапотоков и загрузки/скачивания больших файлов
- **Мультиплексирование соединений**: возможность одновременного установления нескольких двунаправленных и однонаправленных потоков данных
- **Датаграммная связь**: поддержка передачи датаграмм без гарантии порядка и надёжности, идеально подходит для сценариев с критически важными требованиями к реальному времени

По сравнению с WebSocket, WebTransport обеспечивает меньшую задержку и более гибкие режимы связи, особенно эффективно работая в нестабильных сетевых условиях.

## Реализация в Salvo

Фреймворк Salvo предоставляет встроенную поддержку WebTransport, позволяя разработчикам легко создавать приложения на его основе. Ключевые возможности включают:

- Поддержку установки сессий WebTransport
- Поддержку двунаправленных потоков (Bidirectional Streams)
- Поддержку однонаправленных потоков (Unidirectional Streams)
- Поддержку передачи датаграмм (Datagrams)
- Возможность инициации потоков связи сервером

### Конфигурация и запуск

Для запуска приложения Salvo с поддержкой WebTransport требуется настроить TLS-сертификат и QUIC-слушатель:

```rust
let cert = include_bytes!("../certs/cert.pem").to_vec();
let key = include_bytes!("../certs/key.pem").to_vec();

// Конфигурация маршрутизации
let router = Router::new().push(Router::with_path("counter").goal(connect));

// Конфигурация TLS
let config = RustlsConfig::new(Keycert::new().cert(cert.as_slice()).key(key.as_slice()));

// Настройка слушателей
let listener = TcpListener::new(("0.0.0.0", 5800)).rustls(config.clone());
let acceptor = QuinnListener::new(config, ("0.0.0.0", 5800))
    .join(listener)
    .bind()
    .await;

// Запуск сервера
Server::new(acceptor).serve(router).await;
```

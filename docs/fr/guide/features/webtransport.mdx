# WebTransport

WebTransport est un protocole de transport réseau basé sur HTTP/3, offrant une communication bidirectionnelle entre client et serveur, tout en garantissant une faible latence, un débit élevé et une sécurité renforcée.

## Cas d'utilisation

WebTransport est particulièrement adapté aux scénarios suivants :

- **Applications en temps réel** : Jeux en ligne, outils de collaboration instantanée, visioconférences nécessitant une communication à faible latence
- **Transfert de fichiers volumineux** : Prise en charge de transmissions à haut débit, idéal pour les flux multimédias et les transferts de fichiers lourds
- **Communication multiplexée** : Permet d'établir simultanément plusieurs flux de données bidirectionnels et unidirectionnels
- **Communication par datagrammes** : Supporte les datagrammes sans garantie d'ordre ni de fiabilité, adapté aux scénarios exigeant une extrême réactivité

Comparé à WebSocket, WebTransport offre une latence réduite et des modes de communication plus flexibles, particulièrement performants dans des environnements réseau instables.

## Implémentation Salvo

Le framework Salvo intègre nativement le support de WebTransport, permettant aux développeurs de créer facilement des applications basées sur ce protocole. Les principales fonctionnalités incluent :

- Établissement de sessions WebTransport
- Communication par flux bidirectionnels (Bidirectional Streams)
- Communication par flux unidirectionnels (Unidirectional Streams)
- Transmission de datagrammes
- Capacité du serveur à initier des flux de communication

### Configuration et démarrage

Le démarrage d'une application Salvo avec WebTransport nécessite la configuration d'un certificat TLS et d'un écouteur QUIC :

```rust
let cert = include_bytes!("../certs/cert.pem").to_vec();
let key = include_bytes!("../certs/key.pem").to_vec();

// Configuration du routage
let router = Router::new().push(Router::with_path("counter").goal(connect));

// Configuration TLS
let config = RustlsConfig::new(Keycert::new().cert(cert.as_slice()).key(key.as_slice()));

// Configuration de l'écouteur
let listener = TcpListener::new(("0.0.0.0", 5800)).rustls(config.clone());
let acceptor = QuinnListener::new(config, ("0.0.0.0", 5800))
    .join(listener)
    .bind()
    .await;

// Démarrage du serveur
Server::new(acceptor).serve(router).await;
```

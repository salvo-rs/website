# HTTPS erzwingen

Der `force-https`-Middleware kann alle Anfragen zwingen, auf das HTTPS-Protokoll umzuleiten.

Wenn diese Middleware auf einen `Router` angewendet wird, wird die Protokollumleitung nur bei Routenübereinstimmung erzwungen. Falls die Seite nicht existiert, erfolgt keine Umleitung.

Häufiger ist jedoch der Wunsch, dass jede Anfrage automatisch umgeleitet wird, selbst wenn keine Route gefunden wird und ein `404`-Fehler zurückgegeben wird. In diesem Fall kann die Middleware dem `Service` hinzugefügt werden. Unabhängig davon, ob eine Anfrage erfolgreich einer Route zugeordnet wird, wird die auf dem `Service` hinzugefügte Middleware immer ausgeführt.

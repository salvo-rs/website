# Compresión de respuesta

Middleware para comprimir el contenido de `Response`.

Ofrece soporte para tres formatos de compresión: `br`, `gzip`, `deflate`. Se pueden configurar prioridades y otros parámetros según las necesidades.

## Compresión de datos en el protocolo HTTP

1. Nivel de formato de archivo: tipos de archivo específicos utilizan algoritmos de compresión especializados
2. Nivel de protocolo HTTP: compresión de recursos de extremo a extremo mediante codificación de contenido
3. Nivel de conexión: compresión de datos entre nodos de conexión HTTP

Para más información sobre compresión HTTP, consulta la [documentación de Mozilla Developer Network](https://developer.mozilla.org/es/docs/Web/HTTP/Compression).

# Промежуточное ПО для логирования

Логирование играет ключевую роль в веб-приложениях, поскольку позволяет:
- Предоставлять детальную информацию о процессе обработки запросов для отслеживания поведения приложения разработчиками
- Помогать в устранении неполадок и отладке, особенно в производственных средах
- Мониторить производительность приложения и использование ресурсов
- Регистрировать шаблоны доступа пользователей и системные исключения
- Соответствовать требованиям аудита безопасности и нормативным стандартам

Salvo предоставляет промежуточное ПО с базовой функциональностью логирования. Если ПО добавляется напрямую в `Router`, оно не будет фиксировать ошибки 404, возвращаемые при отсутствии совпадения запроса с `Router`. Рекомендуется добавлять его в `Service`.

# Перехват паники в запросах

Catch Panic используется для перехвата сбоев, возникающих при обработке запросов во время выполнения программы. Подробный API смотрите в [документации](https://docs.rs/salvo_extra/latest/salvo_extra/catch_panic/index.html).

> **Примечание**: Для использования `CatchPanic` необходимо включить функцию `catch-panic` в `Cargo.toml`:
> ```toml
> salvo= { version = "xxx", features = ["catch-panic"] }
> ```

## Описание промежуточного ПО

`CatchPanic` — это промежуточное ПО для перехвата паники в обработчиках. Когда во время обработки запроса возникает паника, оно перехватывает её и записывает в ответ ошибку 500 Internal Server Error, вместо того чтобы приводить к краху всего сервера.

**Важное замечание**: Это промежуточное ПО должно использоваться первым, чтобы гарантировать перехват паники в других промежуточных слоях или обработчиках.

## Базовое использование

```rust
use salvo_core::prelude::*;
use salvo_extra::catch_panic::CatchPanic;

#[handler]
async fn hello() {
    panic!("panic error!");
}

#[tokio::main]
async fn main() {
    let router = Router::new().hoop(CatchPanic::new()).get(hello);
    let acceptor = TcpListener::new("0.0.0.0:5800").bind().await;
    Server::new(acceptor).serve(router).await;
}
```

## Сравнение с другими фреймворками для быстрого понимания концепции

### Axum

Аналог промежуточного ПО `catch_panic` из `Tower` в Axum:

```rust
use axum::{
    Router,
    routing::get,
    http::StatusCode,
};
use tower::ServiceBuilder;
use tower_http::catch_panic::CatchPanicLayer;

async fn panic_handler() -> &'static str {
    panic!("panic error!");
}

#[tokio::main]
async fn main() {
    let app = Router::new()
        .route("/", get(panic_handler))
        .layer(
            ServiceBuilder::new()
                .layer(CatchPanicLayer::new())
        );
    
    axum::Server::bind(&"0.0.0.0:3000".parse().unwrap())
        .serve(app.into_make_service())
        .await
        .unwrap();
}
```

### Gin (Go)

В фреймворке Gin для языка Go аналогичным является промежуточное ПО Recovery:

```go
package main

import (
    "github.com/gin-gonic/gin"
)

func main() {
    r := gin.Default() // По умолчанию уже включает промежуточное ПО Recovery
    
    r.GET("/", func(c *gin.Context) {
        panic("panic error!")
    })
    
    r.Run(":8080")
}
```

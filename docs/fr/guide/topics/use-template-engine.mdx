# Utilisation des moteurs de templates

import { Tab, Tabs } from '@rspress/core/theme';

Salvo n'intègre aucun moteur de template par défaut, car les préférences en matière de style de moteur de template varient d'une personne à l'autre.

Fondamentalement, un moteur de template repose sur le principe : données + template = chaîne de caractères.

Ainsi, tant qu'il est possible de produire la chaîne de caractères finale, n'importe quel moteur de template peut être pris en charge.

Par exemple, pour la prise en charge d'`askama` :

<Tabs>
  <Tab label="main.rs">
  ```rust file="<root>/codes/template-askama/src/main.rs"
  ```
  </Tab>
  <Tab label="Cargo.toml">
  ```toml file="<root>/codes/template-askama/Cargo.toml"
  ```
  </Tab>
  <Tab label="templates/hello.html">
  ```html file="<root>/codes/template-askama/templates/hello.html"
  ```
  </Tab>
</Tabs>

**Remarque** : Pour les projets qui ne sont pas particulièrement complexes, nous recommandons plutôt une approche de développement avec séparation frontend/backend. Utilisez des frameworks UI plus flexibles et bénéficiant d'un meilleur écosystème (comme React, Vue, Svelte, etc.) pour construire le frontend, tandis que Salvo sert de service backend d'API. Cette méthode offre une meilleure efficacité de développement, une séparation claire des responsabilités entre frontend et backend, et correspond davantage aux tendances actuelles de développement d'applications web modernes.

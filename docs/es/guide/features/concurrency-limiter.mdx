# Limitación de Concurrencia

El middleware **Concurrency Limiter** permite controlar el número de solicitudes procesadas simultáneamente. Consulte la [documentación de la API](https://docs.rs/salvo_extra/latest/salvo_extra/concurrency_limiter/index.html) para más detalles.

## Casos de Uso

Este limitador es especialmente útil en:

- **Protección de sitios con alto tráfico**: Limita solicitudes concurrentes durante picos de tráfico para evitar sobrecarga del servidor.
- **Operaciones intensivas en recursos**: Controla la concurrencia en operaciones como subida de archivos o escrituras en bases de datos.
- **Llamadas a APIs externas con límites**: Regula solicitudes concurrentes cuando se interactúa con servicios que imponen restricciones de frecuencia.

## Beneficios Clave

- **Previene sobrecarga del servidor**: Mantiene la estabilidad durante tráfico elevado.
- **Mejora la confiabilidad**: Evita fallos por agotamiento de recursos.
- **Optimiza recursos**: Garantiza disponibilidad para servicios críticos.

## Implementación

Añada el middleware `max_concurrency(n)` a su ruta, donde `n` es el máximo de solicitudes permitidas. Las peticiones excedentes recibirán respuesta `503 Service Unavailable`.

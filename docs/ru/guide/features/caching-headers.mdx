# Заголовки кеширования

Промежуточное ПО, предоставляющее поддержку настройки заголовков кеширования.

Управление кешированием является важной частью оптимизации веб-производительности. Правильная настройка заголовков кеширования позволяет уменьшить количество ненужных сетевых запросов и повысить производительность приложения. [Cache-Control](https://developer.mozilla.org/ru/docs/Web/HTTP/Headers/Cache-Control) — это HTTP-заголовок ответа, содержащий директивы для стратегии кеширования в браузере. Он определяет, кто и при каких условиях может кешировать ответ, а также срок действия кеша.

Фактически реализация включает три обработчика: `CachingHeaders`, `Modified` и `ETag`, где `CachingHeaders` является комбинацией двух последних. В обычных случаях рекомендуется использовать `CachingHeaders`.

- `Modified`: Обеспечивает проверку кеша на основе времени последнего изменения ресурса.
- `ETag`: Использует тег сущности (Entity Tag) для более точного механизма проверки ресурса.
- `CachingHeaders`: Объединяет оба механизма, предоставляя полную поддержку управления кешированием.

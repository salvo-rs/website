# Response Compression

A middleware for compressing the content of `Response`.

Supports three compression formats: `br`, `gzip`, and `deflate`. Allows configuration of compression priorities and other parameters as needed.

## Data Compression in HTTP Protocol

1. File format level: Specific file types use dedicated compression algorithms  
2. HTTP protocol level: End-to-end resource compression transmission through content encoding  
3. Connection level: Data compression between nodes of an HTTP connection  

For more information about HTTP compression, refer to [Mozilla Developer Network documentation](https://developer.mozilla.org/en-US/docs/Web/HTTP/Compression).

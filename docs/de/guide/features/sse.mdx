## Was ist SSE?

Server-Sent Events (SSE) ist eine Webtechnologie, die es Servern ermöglicht, Daten an Clients zu pushen. Im Gegensatz zu WebSocket ist SSE eine unidirektionale Kommunikation, bei der Daten nur vom Server zum Client gesendet werden können. Es eignet sich für Szenarien, in denen der Server Echtzeitdaten pushen muss, der Client jedoch keine häufigen Daten senden muss. SSE basiert auf dem HTTP-Protokoll, ist einfach zu implementieren und verfügt über einen integrierten Mechanismus zur Wiederherstellung der Verbindung bei Unterbrechungen.

## Integration mit großen Sprachmodellen

Bei der Integration von KI über das OpenAI SDK oder andere SDKs für große Sprachmodelle ist die Rückgabe von Streaming-Ergebnissen (Stream) ein häufiges Anforderungsszenario. SSE bietet eine perfekte Lösung, um KI-generierte Inhalte in Echtzeit an das Frontend zu pushen und so eine bessere Benutzererfahrung zu ermöglichen.

Beispielsweise kann bei der Nutzung der Chat Completion API von OpenAI mit Streaming-Antworten die SSE-Funktion von Salvo genutzt werden, um KI-generierte Inhalte schrittweise an den Client zu senden und so die Verzögerung zu vermeiden, die durch das Warten auf eine vollständige Antwort entsteht.

# Написание тестов

## Важность тестирования

Написание тестов — это проявление ответственности инженера и секрет спокойного сна. Полноценный набор тестов не только повышает качество кода и предотвращает регрессионные ошибки, но и позволяет спать спокойно даже после развертывания. Когда ваши коллеги посреди ночи получают оповещения о сбое системы, ваше приложение остается непоколебимым — вот спокойствие и уверенность, которые дают тесты.

## Инструменты тестирования Salvo

Модуль `test`, предоставляемый Salvo, помогает тестировать проекты на основе Salvo.

[Актуальная документация](https://docs.rs/salvo_core/latest/salvo_core/test/index.html)

**Простой пример:**

```rust
use salvo::prelude::*;

#[handler]
async fn hello_world() -> &'static str {
    "Hello World"
}

#[tokio::main]
async fn main() {
    tracing_subscriber::fmt().init();

    tracing::info!("Listening on http://127.0.0.1:5800");
    let acceptor = TcpListener::new("127.0.0.1:5800").bind().await; Server::new(acceptor).serve(route()).await;
}

fn route() -> Router {
    Router::new().get(hello_world)
}

#[cfg(test)]
mod tests {
    use salvo::prelude::*;
    use salvo::test::{ResponseExt, TestClient};

    #[tokio::test]
    async fn test_hello_world() {
        let service = Service::new(super::route());

        let content = TestClient::get(format!("http://127.0.0.1:5800/"))
            .send(&service)
            .await
            .take_string()
            .await
            .unwrap();
        assert_eq!(content, "Hello World");
    }
}
```

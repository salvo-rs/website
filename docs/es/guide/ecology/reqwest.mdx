# Reqwest: Biblioteca de cliente HTTP en Rust

[Reqwest](https://docs.rs/reqwest/latest/reqwest/) es una biblioteca de cliente HTTP de alto nivel que simplifica el proceso de manejo de solicitudes HTTP, proporcionando muchas funciones comunes:

- Soporte para API asíncronas y bloqueantes
- Manejo de varios tipos de cuerpos de solicitud: texto plano, JSON, formularios codificados en URL, formularios multiparte
- Política de redirección personalizable
- Soporte para proxy HTTP
- Cifrado TLS por defecto
- Gestión de cookies

## Uso básico

### Realizar una solicitud GET

Para una sola solicitud, se puede usar el método abreviado `get`:

```rust
let body = reqwest::get("https://www.rust-lang.org")
    .await?
    .text()
    .await?;

println!("body = {body:?}");
```

> Nota: Si planeas realizar múltiples solicitudes, es mejor crear un `Client` y reutilizarlo para aprovechar las ventajas del grupo de conexiones.

```rust
let client = reqwest::Client::new();
let res = client.get("https://www.rust-lang.org")
    .send()
    .await?;
```

### Realizar una solicitud POST

Se puede establecer el cuerpo de la solicitud usando el método `body()`:

```rust
let client = reqwest::Client::new();
let res = client.post("http://httpbin.org/post")
    .body("el contenido exacto a enviar")
    .send()
    .await?;
```

### Datos de formulario

Enviar datos de formulario es una necesidad común, se puede usar cualquier tipo que sea serializable a datos de formulario:

```rust
// Esto enviará una solicitud POST con el cuerpo `foo=bar&baz=quux`
let params = [("foo", "bar"), ("baz", "quux")];
let client = reqwest::Client::new();
let res = client.post("http://httpbin.org/post")
    .form(&params)
    .send()
    .await?;
```

### Datos JSON

Se pueden enviar fácilmente datos JSON mediante el método `json` (requiere la característica `json`):

```rust
// Esto enviará una solicitud POST con el cuerpo `{"lang":"rust","body":"json"}`
let mut map = HashMap::new();
map.insert("lang", "rust");
map.insert("body", "json");

let client = reqwest::Client::new();
let res = client.post("http://httpbin.org/post")
    .json(&map)
    .send()
    .await?;
```

### Manejo de respuestas

Se puede manejar la respuesta de varias maneras:

```rust
let res = client.get("https://www.rust-lang.org").send().await?;

// Obtener el código de estado
let status = res.status();

// Obtener las cabeceras de respuesta
let content_type = res.headers().get("content-type").unwrap();

// Leer el cuerpo de la respuesta como texto
let body = res.text().await?;

// O analizarlo como JSON
let json: serde_json::Value = res.json().await?;
```

## Funciones avanzadas

### Política de redirección

Por defecto, el cliente maneja automáticamente las redirecciones HTTP, siguiendo hasta 10 saltos. Se puede personalizar este comportamiento usando `ClientBuilder`:

```rust
let custom_client = reqwest::Client::builder()
    .redirect(reqwest::redirect::Policy::none()) // Deshabilitar redirección
    .build()?;
```

### Soporte para cookies

Se puede habilitar el almacenamiento y envío automático de cookies de sesión a través de `ClientBuilder`:

```rust
let client = reqwest::Client::builder()
    .cookie_store(true)
    .build()?;
```

### Configuración de proxy

El proxy del sistema está habilitado por defecto, buscando configuraciones de proxy HTTP o HTTPS en variables de entorno:

- `HTTP_PROXY` o `http_proxy`: proxy para conexiones HTTP
- `HTTPS_PROXY` o `https_proxy`: proxy para conexiones HTTPS
- `ALL_PROXY` o `all_proxy`: proxy para ambos tipos de conexiones

También se puede configurar explícitamente un proxy mediante código:

```rust
let proxy = reqwest::Proxy::http("https://secure.example")?;
let client = reqwest::Client::builder()
    .proxy(proxy)
    .build()?;

// O deshabilitar el proxy
let client = reqwest::Client::builder()
    .no_proxy()
    .build()?;
```

### Configuración TLS

El cliente usa TLS por defecto para conectarse a destinos HTTPS:

```rust
// Agregar un certificado de servidor adicional
let cert = reqwest::Certificate::from_pem(&cert_bytes)?;
let client = reqwest::Client::builder()
    .add_root_certificate(cert)
    .build()?;

// Configurar un certificado de cliente
let identity = reqwest::Identity::from_pkcs12_der(&pkcs12_der, "password")?;
let client = reqwest::Client::builder()
    .identity(identity)
    .build()?;
```

### Configuración de tiempo de espera

Se pueden configurar tiempos de espera para las solicitudes:

```rust
let client = reqwest::Client::builder()
    .timeout(std::time::Duration::from_secs(10))
    .build()?;
```

## Características opcionales

Reqwest proporciona varias características opcionales que se pueden habilitar o deshabilitar a través de características de Cargo:

- `http2` (habilitado por defecto): soporte para HTTP/2
- `default-tls` (habilitado por defecto): proporciona soporte TLS para HTTPS
- `rustls-tls`: proporciona funcionalidad TLS usando rustls
- `blocking`: proporciona API de cliente bloqueante
- `json`: proporciona funcionalidad de serialización y deserialización JSON
- `multipart`: proporciona funcionalidad de formularios multiparte
- `cookies`: proporciona soporte para sesiones de cookies
- `gzip`, `brotli`, `deflate`, `zstd`: soporte para varias descompresiones de cuerpos de respuesta
- `socks`: proporciona soporte para proxy SOCKS5

## API bloqueante

Cuando no se necesitan operaciones asíncronas, se puede usar la API bloqueante (requiere la característica `blocking`):

```rust
let body = reqwest::blocking::get("https://www.rust-lang.org")?.text()?;

let client = reqwest::blocking::Client::new();
let res = client.post("http://httpbin.org/post")
    .json(&map)
    .send()?;
```

# Logging-Middleware

Logging spielt eine entscheidende Rolle in Webanwendungen, da es Folgendes ermöglicht:
- Bereitstellung detaillierter Informationen zum Anfrageverarbeitungsprozess, um Entwicklern bei der Nachverfolgung des Anwendungsverhaltens zu helfen
- Unterstützung bei der Fehlerbehebung und Fehlersuche, insbesondere in Produktionsumgebungen
- Überwachung der Anwendungsleistung und Ressourcennutzung
- Aufzeichnung von Benutzerzugriffsmustern und Systemausnahmen
- Erfüllung von Sicherheitsaudit- und Compliance-Anforderungen

Salvo bietet eine Middleware mit grundlegender Logging-Funktionalität. Wenn die Middleware direkt zu einem `Router` hinzugefügt wird, werden 404-Fehler, die zurückgegeben werden, wenn keine Übereinstimmung mit einem `Router` gefunden wird, nicht erfasst. Es wird empfohlen, sie stattdessen zum `Service` hinzuzufügen.

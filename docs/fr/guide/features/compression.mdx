# Compression de Réponse

Un middleware pour compresser le contenu de la `Réponse`.

Prend en charge trois formats de compression : `br`, `gzip` et `deflate`. La priorité et la configuration de chaque méthode de compression peuvent être personnalisées selon les besoins.

## Compression des données dans le protocole HTTP

1. Niveau format de fichier : Les types de fichiers spécifiques utilisent des algorithmes de compression dédiés.
2. Niveau protocole HTTP : La transmission de ressources compressées de bout en bout est réalisée via l'encodage du contenu.
3. Niveau connexion : La compression des données est effectuée entre les nœuds d'une connexion HTTP.

Pour plus d'informations sur la compression HTTP, consultez la [documentation du Mozilla Developer Network](https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Compression).

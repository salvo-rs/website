# Совместимость с промежуточным ПО Tower

import { Tab, Tabs } from '@rspress/core/theme';

Salvo обеспечивает поддержку совместимости с экосистемой Tower через функцию `tower-compat`. Подробную информацию об API смотрите в [документации](https://docs.rs/salvo_extra/latest/salvo_extra/tower_compat/index.html).

## Основные концепции Tower

Tower — это библиотека абстракций сервисов, основанная на двух ключевых трейтах:

### Трейт Service

```rust
pub trait Service<Request> {
    type Response;
    type Error;
    type Future: Future<Output = Result<Self::Response, Self::Error>>;

    fn poll_ready(&mut self, cx: &mut Context<'_>) -> Poll<Result<(), Self::Error>>;
    fn call(&mut self, req: Request) -> Self::Future;
}
```

`Service` отвечает за обработку запросов и возврат ответов, аналогично обработчикам в Salvo.

### Трейт Layer

```rust
pub trait Layer<S> {
    type Service;
    fn layer(&self, inner: S) -> Self::Service;
}
```

`Layer` используется для обёртывания сервисов и добавления дополнительной функциональности, аналогично промежуточному ПО в Salvo.

## Поддержка совместимости в Salvo

Salvo предоставляет две ключевые реализации трейтов для обеспечения совместимости с экосистемой Tower:

- **TowerLayerCompat**: преобразует `Layer` из Tower в `Handler` Salvo, позволяя использовать его как Hoop (промежуточное ПО)
- **TowerServiceCompat**: преобразует `Service` из Tower в `Handler` Salvo

Пример использования:
```rust
// Использование RateLimitLayer из Tower с преобразованием в совместимый формат Salvo через метод compat()
let limit = RateLimitLayer::new(5, Duration::from_secs(30)).compat();
let router = Router::new().hoop(limit).get(hello);
```

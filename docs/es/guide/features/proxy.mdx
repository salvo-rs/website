# Proxy Inverso

Un proxy inverso es una arquitectura de servidor que recibe solicitudes de clientes y las reenvía a uno o más servidores backend. A diferencia de un proxy directo (que actúa en nombre de los clientes), un proxy inverso opera en representación del lado del servidor.

Ventajas clave de los proxies inversos:
- **Balanceo de Carga**: Distribuye solicitudes entre múltiples servidores
- **Seguridad Mejorada**: Oculta la información real del servidor
- **Almacenamiento en Caché de Contenido**: Mejora el rendimiento
- **Reescritura y Reenvío de Rutas**: Enruta solicitudes de manera flexible

El framework Salvo proporciona middleware para funcionalidad de proxy inverso.

_**Código de Ejemplo**_

```rust
use salvo::prelude::*;
use salvo::proxy::Proxy;

#[tokio::main]
async fn main() {
    tracing_subscriber::fmt().init();
    
    let router = Router::new()
        .push(
            Router::new()
                .path("google/{**rest}")
                .handle(Proxy::<Vec<&str>>::new(vec!["https://www.google.com"])),
        )
        .push(
            Router::new()
                .path("baidu/{**rest}")
                .handle(Proxy::<Vec<&str>>::new(vec!["https://www.baidu.com"])),
        );
    let acceptor = TcpListener::new("127.0.0.1:5800").bind().await; Server::new(acceptor).serve(router).await;
}
```

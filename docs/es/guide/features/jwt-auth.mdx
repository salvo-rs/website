# Autenticación JWT

JWT (JSON Web Token) es un estándar abierto (RFC 7519) para transmitir información de forma segura entre partes. Es un método compacto y seguro para URL que representa afirmaciones en formato de objeto JSON, comúnmente utilizado para autenticación e intercambio de información.

Un JWT consta de tres partes:
- **Encabezado** - Especifica el tipo de token y el algoritmo de firma utilizado
- **Carga útil** - Contiene las afirmaciones, como ID de usuario, roles, tiempo de expiración, etc.
- **Firma** - Se utiliza para verificar que el mensaje no fue alterado durante la transmisión

Uso típico de JWT en el flujo de autenticación:
1. Después del inicio de sesión del usuario, el servidor genera un token JWT
2. El token se devuelve al cliente y se almacena (normalmente en localStorage o cookies)
3. En solicitudes posteriores, el cliente incluye este token en el encabezado de Autorización
4. El servidor valida la autenticidad del token y concede acceso

Proporciona middleware para autenticación JWT Auth.

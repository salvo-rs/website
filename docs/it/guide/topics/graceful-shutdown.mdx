# Arresto Elegante

L'arresto elegante si riferisce alla modalità di chiusura del server in cui non tutte le connessioni vengono terminate immediatamente, ma si interrompe prima l'accettazione di nuove richieste, concedendo alle richieste già accettate il tempo necessario per completare l'elaborazione prima di chiudere il servizio. Questo approccio evita l'interruzione improvvisa delle richieste, migliorando l'esperienza utente e l'affidabilità del sistema.

Salvo offre supporto per l'arresto elegante attraverso il metodo `handle` del `Server`, che restituisce un handle del server, seguito dalla chiamata al metodo `stop_graceful`. Dopo l'invocazione di questo metodo, il server:

- Cessa di accettare nuove connessioni
- Attende il completamento delle richieste in corso
- Forza la chiusura delle connessioni rimanenti dopo il timeout specificato (se fornito)

Ecco un esempio semplice:

```rust
use salvo_core::prelude::*;

#[tokio::main]
async fn main() {
    let acceptor = TcpListener::new("127.0.0.1:5800").bind().await;
    let server = Server::new(acceptor);
    let handle = server.handle();

    // Arresto elegante del server
    tokio::spawn(async move {
        tokio::time::sleep(std::time::Duration::from_secs(60)).await;
        handle.stop_graceful(None);
    });
    server.serve(Router::new()).await;
}
```

Nell'esempio sopra:

- `server.handle()` ottiene l'handle del server, utilizzabile per controllarne il ciclo di vita
- `handle.stop_graceful(None)` avvia il processo di arresto elegante; `None` indica l'assenza di timeout, attendendo indefinitamente il completamento di tutte le richieste
- Per impostare un timeout, è possibile passare `Some(Duration)`; allo scadere del timeout, le connessioni residue verranno forzatamente chiuse

Questo approccio è particolarmente adatto per applicazioni distribuite in ambienti containerizzati o su piattaforme cloud, nonché per garantire che le richieste non vengano interrotte accidentalmente durante aggiornamenti a caldo.

# Usando Mecanismos de Modelagem

O Salvo não possui nenhum mecanismo de modelagem embutido, afinal, a preferência por um estilo específico de mecanismo de modelagem varia de pessoa para pessoa.

Em essência, um mecanismo de modelagem é: dados + modelo = string.

Portanto, desde que seja possível renderizar a string final, qualquer mecanismo de modelagem pode ser suportado.

**Nota**: Para projetos não particularmente complexos, recomendamos o uso de uma abordagem de desenvolvimento com separação entre front-end e back-end. Utilize frameworks de UI mais flexíveis e com ecossistema mais robusto (como React, Vue, Svelte, etc.) para construir o front-end, enquanto o Salvo atua como serviço de API no back-end. Essa abordagem aumenta a eficiência de desenvolvimento, define responsabilidades mais claras entre front-end e back-end e está mais alinhada com as tendências modernas de desenvolvimento de aplicações web.

# Proxy Inverso

Un proxy inverso es una arquitectura de servidor que recibe solicitudes de clientes y las reenvía a uno o varios servidores backend. A diferencia de un proxy directo (que actúa en nombre del cliente), el proxy inverso trabaja en representación del lado del servidor.

Principales ventajas del proxy inverso:
- **Balanceo de carga**: Distribuye solicitudes entre múltiples servidores
- **Mayor seguridad**: Oculta la información del servidor real
- **Caché de contenido**: Mejora el rendimiento
- **Reescritura y redirección de rutas**: Enruta solicitudes de manera flexible

El framework Salvo proporciona middleware con funcionalidad de proxy inverso.

_**Código de ejemplo**_

```rust
use salvo::prelude::*;
use salvo::proxy::Proxy;

#[tokio::main]
async fn main() {
    tracing_subscriber::fmt().init();
    
    let router = Router::new()
        .push(
            Router::new()
                .path("google/{**rest}")
                .handle(Proxy::<Vec<&str>>::new(vec!["https://www.google.com"])),
        )
        .push(
            Router::new()
                .path("baidu/{**rest}")
                .handle(Proxy::<Vec<&str>>::new(vec!["https://www.baidu.com"])),
        );
    let acceptor = TcpListener::new("127.0.0.1:5800").bind().await; Server::new(acceptor).serve(router).await;
}
```

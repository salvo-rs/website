Middleware que fornece suporte a `Session`.

## O que é Session

Session é um conjunto de dados do usuário armazenados no lado do servidor, geralmente usados para rastrear o estado do usuário e salvar dados temporários. Em aplicações web, a Session resolve o problema da falta de estado no protocolo HTTP, permitindo que o servidor identifique múltiplas solicitações do mesmo usuário.

## Uso de Session no Salvo

No Salvo, o middleware Session fornece uma API simples para ler e escrever dados de sessão. Através do `SessionDepot`, é possível obter uma instância da sessão, e métodos como `get`, `set`, `remove` permitem gerenciar os dados da sessão.

## Usos Comuns

- Autenticação de usuário e gerenciamento de estado de login
- Armazenamento temporário de dados de formulário
- Armazenamento de preferências do usuário

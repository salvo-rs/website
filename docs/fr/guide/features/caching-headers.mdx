# En-têtes de cache

Middleware fournissant la prise en charge de la configuration des en-têtes de cache.

Le contrôle du cache est une partie essentielle de l'optimisation des performances web. En configurant correctement les en-têtes de cache, on peut réduire les requêtes réseau inutiles et améliorer les performances de l'application. [Cache-Control](https://developer.mozilla.org/fr/docs/Web/HTTP/Headers/Cache-Control) est un en-tête de réponse HTTP qui spécifie les directives de stratégie de cache du navigateur, contrôlant qui peut mettre en cache la réponse, sous quelles conditions et pour combien de temps.

L'implémentation interne comprend trois gestionnaires (`Handler`) : `CachingHeaders`, `Modified` et `ETag`. `CachingHeaders` est une combinaison des deux derniers. En usage normal, on utilise `CachingHeaders`.

- `Modified` : Fournit une validation de cache basée sur la date de dernière modification de la ressource.
- `ETag` : Utilise une balise d'entité (Entity Tag) pour offrir un mécanisme de validation de ressource plus précis.
- `CachingHeaders` : Combine les deux mécanismes ci-dessus pour fournir une prise en charge complète du contrôle de cache.

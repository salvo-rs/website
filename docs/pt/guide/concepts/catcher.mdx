# Catcher

import { Tab, Tabs } from '@rspress/core/theme';

Quando uma `Response` retorna um código de status de erro e o `Body` dentro da página está vazio, o Salvo tentará capturar esse erro usando um `Catcher` e exibir uma página de erro amigável ao usuário.

Você pode obter um `Catcher` padrão do sistema chamando `Catcher::default()`, e então adicioná-lo ao `Service`.

```rust
use salvo::catcher::Catcher;

Service::new(router).catcher(Catcher::default());
```

O `Catcher` padrão suporta o envio de páginas de erro nos formatos `XML`, `JSON`, `HTML` e `Text`.

Você pode adicionar manipuladores personalizados de captura de erro ao `Catcher` anexando `hoop`s a este `Catcher` padrão. Esses manipuladores de captura de erro ainda são do tipo `Handler`.

Você pode adicionar vários manipuladores personalizados de captura de erro ao `Catcher` através de `hoop`s. Os manipuladores de erro personalizados podem chamar o método `FlowCtrl::skip_next` após processar um erro para pular os manipuladores de erro subsequentes e retornar antecipadamente.

# Возможности Craft

Craft позволяет разработчикам автоматически генерировать обработчики и конечные точки с помощью простых аннотаций, обеспечивая при этом бесшовную интеграцию с генерацией документации OpenAPI.

## Сценарии использования

Возможности Craft особенно полезны в следующих случаях:

- Когда необходимо быстро создать обработчики маршрутов из методов структур
- Когда требуется сократить шаблонный код для извлечения параметров и обработки ошибок
- Когда нужно автоматически генерировать документацию OpenAPI для вашего API
- Когда требуется отделить бизнес-логику от веб-фреймворка

## Базовое использование

Для использования возможностей Craft необходимо импортировать следующие модули:

```rust
use salvo::oapi::extract::*;
use salvo::prelude::*;
```

### Создание структуры сервиса

Используйте макрос `#[craft]` для аннотации блока impl, чтобы преобразовать методы структуры в обработчики или конечные точки.

```rust
#[derive(Clone)]
pub struct Opts {
    state: i64,
}

#[craft]
impl Opts {
    // Конструктор
    fn new(state: i64) -> Self {
        Self { state }
    }
    
    // Дополнительные методы...
}
```

### Создание обработчика

Используйте `#[craft(handler)]` для преобразования метода в обработчик:

```rust
#[craft(handler)]
fn add1(&self, left: QueryParam<i64>, right: QueryParam<i64>) -> String {
    (self.state + *left + *right).to_string()
}
```

Этот метод станет обработчиком, который:
- Автоматически извлекает значения `left` и `right` из параметров запроса
- Получает доступ к состоянию `state` из структуры
- Возвращает результат вычисления в виде строкового ответа

### Создание конечной точки

Используйте `#[craft(endpoint)]` для преобразования метода в конечную точку:

```rust
#[craft(endpoint)]
pub(crate) fn add2(
    self: ::std::sync::Arc<Self>,
    left: QueryParam<i64>,
    right: QueryParam<i64>,
) -> String {
    (self.state + *left + *right).to_string()
}
```

Конечные точки могут использовать `Arc` для совместного использования состояния, что особенно полезно при обработке параллельных запросов.

### Статические конечные точки

Вы также можете создавать статические конечные точки, не зависящие от состояния экземпляра:

```rust
#[craft(endpoint(responses((status_code = 400, description = "Wrong request parameters."))))]
pub fn add3(left: QueryParam<i64>, right: QueryParam<i64>) -> String {
    (*left + *right).to_string()
}
```

В этом примере также добавлено описание пользовательского ответа об ошибке, которое будет отражено в сгенерированной документации OpenAPI.

## Извлекатели параметров

Модуль `oapi::extract` в Salvo предоставляет различные извлекатели параметров, наиболее распространенные из которых включают:

- `QueryParam<T>`: извлекает параметры из строки запроса
- `PathParam<T>`: извлекает параметры из пути URL
- `FormData<T>`: извлекает параметры из данных формы
- `JsonBody<T>`: извлекает параметры из JSON-тела запроса

Эти извлекатели автоматически выполняют разбор параметров и преобразование типов, значительно упрощая написание обработчиков.

## Интеграция с OpenAPI

Возможности Craft автоматически генерируют документацию API, соответствующую спецификации OpenAPI. В примере:

```rust
let router = Router::new()
    .push(Router::with_path("add1").get(opts.add1()))
    .push(Router::with_path("add2").get(opts.add2()))
    .push(Router::with_path("add3").get(Opts::add3()));

// Генерация документации OpenAPI
let doc = OpenApi::new("Example API", "0.0.1").merge_router(&router);

// Добавление маршрутов для документации OpenAPI и Swagger UI
let router = router
    .push(doc.into_router("/api-doc/openapi.json"))
    .push(SwaggerUi::new("/api-doc/openapi.json").into_router("swagger-ui"));
```

После такой настройки документация API будет доступна по конечной точке `/api-doc/openapi.json`, а Swagger UI — по пути `/swagger-ui`.

## Полный пример

Ниже приведен полный пример, демонстрирующий использование возможностей Craft для создания трех различных типов конечных точек:

```rust
use salvo::oapi::extract::*;
use salvo::prelude::*;
use std::sync::Arc;

#[derive(Clone)]
pub struct Opts {
    state: i64,
}

#[craft]
impl Opts {
    fn new(state: i64) -> Self {
        Self { state }
    }

    #[craft(handler)]
    fn add1(&self, left: QueryParam<i64>, right: QueryParam<i64>) -> String {
        (self.state + *left + *right).to_string()
    }

    #[craft(endpoint)]
    pub(crate) fn add2(
        self: ::std::sync::Arc<Self>,
        left: QueryParam<i64>,
        right: QueryParam<i64>,
    ) -> String {
        (self.state + *left + *right).to_string()
    }

    #[craft(endpoint(responses((status_code = 400, description = "Wrong request parameters."))))]
    pub fn add3(left: QueryParam<i64>, right: QueryParam<i64>) -> String {
        (*left + *right).to_string()
    }
}

#[tokio::main]
async fn main() {
    // Создание общего состояния с начальным значением 1
    let opts = Arc::new(Opts::new(1));

    // Настройка маршрутов для трех конечных точек
    let router = Router::new()
        .push(Router::with_path("add1").get(opts.add1()))
        .push(Router::with_path("add2").get(opts.add2()))
        .push(Router::with_path("add3").get(Opts::add3()));

    // Генерация документации OpenAPI
    let doc = OpenApi::new("Example API", "0.0.1").merge_router(&router);
    
    // Добавление маршрутов для документации OpenAPI и Swagger UI
    let router = router
        .push(doc.into_router("/api-doc/openapi.json"))
        .push(SwaggerUi::new("/api-doc/openapi.json").into_router("swagger-ui"));

    // Запуск сервера на localhost:5800
    let acceptor = TcpListener::new("127.0.0.1:5800").bind().await;
    Server::new(acceptor).serve(router).await;
}
```

Middleware que proporciona soporte para Basic Auth.

## Introducción a Basic Auth

Basic Auth es un mecanismo simple de autenticación HTTP que transmite el nombre de usuario y contraseña a través del encabezado de la solicitud HTTP en el campo `Authorization`. El formato es `Authorization: Basic <base64(usuario:contraseña)>`. Aunque es simple, como las credenciales solo están codificadas en Base64 y no encriptadas, generalmente debe usarse junto con HTTPS para garantizar la seguridad.

## Comparación de implementaciones de Basic Auth en frameworks comunes

| Framework | Lenguaje | Implementación de Basic Auth |
|-----------|----------|------------------------------|
| Salvo | Rust | Implementado mediante el middleware `BasicAuth` y un `BasicAuthValidator` personalizado |
| Express | JavaScript | Usa el paquete de middleware `express-basic-auth` |
| Spring Security | Java | Configurado mediante `httpBasic()` e implementado con `UserDetailsService` |
| ASP.NET Core | C# | Usa `.UseAuthentication()` y `AddAuthentication(AuthenticationSchemes.Basic)` |
| Gin | Go | Mediante middleware personalizado o usando el paquete `gin-contrib/auth` |

## Casos de uso

Basic Auth es adecuado para los siguientes escenarios:

1. **APIs y herramientas internas**: Herramientas de administración y APIs utilizadas internamente en la empresa.
2. **Entornos de desarrollo y pruebas**: Para implementar autenticación rápidamente sin necesidad de un sistema de inicio de sesión complejo.
3. **Protección simple de APIs**: Cuando no se requiere un sistema complejo de gestión de usuarios.
4. **En combinación con otras medidas de seguridad**: Como parte de una arquitectura de seguridad multicapa.

En Salvo, el middleware de Basic Auth se puede integrar fácilmente en las rutas. Es muy flexible, ya que permite personalizar la lógica de validación implementando el trait `BasicAuthValidator`.

## Consideraciones

- Siempre debe usarse junto con HTTPS para proteger la transmisión de credenciales.
- No es adecuado para entornos de producción que almacenen información sensible.
- Considere usar métodos de autenticación más seguros como JWT, OAuth, etc., para entornos de producción.

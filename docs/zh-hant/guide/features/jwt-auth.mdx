# JWT 驗證

JWT（JSON Web Token）是一種開放標準（RFC 7519），用於在各方之間安全地傳輸資訊。它採用緊湊且URL安全的格式，以JSON物件形式承載聲明內容，通常用於身份驗證與資訊交換。

JWT由三部分組成：
- **標頭（Header）** - 指定權杖類型與使用的簽章演算法
- **載荷（Payload）** - 包含聲明內容，例如使用者ID、角色、過期時間等
- **簽章（Signature）** - 用於驗證訊息在傳輸過程中是否遭竄改

JWT在身份驗證流程中的典型應用方式：
1. 使用者登入後，伺服器產生JWT權杖
2. 權杖傳回客戶端並儲存（通常置於localStorage或cookie中）
3. 後續請求中，客戶端透過Authorization標頭攜帶此權杖
4. 伺服器驗證權杖有效性並授予存取權限

# Uso de motores de plantillas

import { Tab, Tabs } from '@rspress/core/theme';

Salvo no incluye ningún motor de plantillas integrado, ya que la preferencia por un estilo particular de motor de plantillas varía según cada persona.

En esencia, un motor de plantillas es: datos + plantilla = cadena de texto.

Por lo tanto, cualquier motor de plantillas puede ser compatible siempre que pueda renderizar la cadena de texto final.

Por ejemplo, el soporte para `askama`:

**Nota**: Para proyectos que no sean especialmente complejos, recomendamos utilizar un enfoque de desarrollo con separación entre frontend y backend. Esto implica emplear frameworks de UI más flexibles y con mejor ecosistema (como React, Vue, Svelte, etc.) para construir el frontend, mientras que Salvo actúa como servicio de API en el backend. Este método mejora la eficiencia del desarrollo, clarifica las responsabilidades entre frontend y backend, y se alinea mejor con las tendencias actuales en el desarrollo de aplicaciones web.

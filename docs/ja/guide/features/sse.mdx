## SSEとは

Server-Sent Events (SSE) は、サーバーがクライアントにデータをプッシュすることを可能にするWeb技術です。WebSocketとは異なり、SSEは単方向通信であり、サーバーからクライアントへのデータ送信のみ可能です。これは、サーバーからのリアルタイムデータプッシュが必要で、クライアントが頻繁にデータを送信する必要がないシナリオに適しています。SSEはHTTPプロトコルに基づいており、実装が簡単で、自動的な再接続メカニズムを備えています。

## 大規模言語モデルとの統合

OpenAI SDKや他の大規模言語モデルSDKを使用してAIを統合する際、ストリーミングによる結果の返却（Stream）は一般的な要件です。SSEは完璧な解決策を提供し、AIが生成するコンテンツをリアルタイムでフロントエンドにプッシュし、より優れたユーザー体験を実現します。

例えば、OpenAIのChat Completion APIを統合し、ストリーミングレスポンスを使用する場合、SalvoのSSE機能を活用してAIが生成するコンテンツを段階的にクライアントに送信し、ユーザーが完全なレスポンスを待つ際の遅延感を解消できます。

# Compatibilidade de Middleware Tower

import { Tab, Tabs } from '@rspress/core/theme';

O Salvo fornece suporte de compatibilidade para o ecossistema Tower através do recurso `tower-compat`. Para APIs específicas, consulte [a documentação](https://docs.rs/salvo_extra/latest/salvo_extra/tower_compat/index.html).

## Conceitos Principais do Tower

Tower é uma biblioteca de abstração de serviços baseada principalmente em dois traços centrais:

### Traço Service

```rust
pub trait Service<Request> {
    type Response;
    type Error;
    type Future: Future<Output = Result<Self::Response, Self::Error>>;

    fn poll_ready(&mut self, cx: &mut Context<'_>) -> Poll<Result<(), Self::Error>>;
    fn call(&mut self, req: Request) -> Self::Future;
}
```

O traço `Service` é responsável por processar requisições e retornar respostas, semelhante aos handlers do Salvo.

### Traço Layer

```rust
pub trait Layer<S> {
    type Service;
    fn layer(&self, inner: S) -> Self::Service;
}
```

O traço `Layer` é usado para encapsular serviços e adicionar funcionalidades extras, semelhante ao middleware do Salvo.

## Suporte de Compatibilidade do Salvo

O Salvo fornece dois traços-chave para alcançar compatibilidade com o ecossistema Tower:

- **TowerLayerCompat**: Converte o `Layer` do Tower em um `Handler` do Salvo, permitindo seu uso como um Hoop (middleware).
- **TowerServiceCompat**: Converte o `Service` do Tower em um `Handler` do Salvo.

Exemplo de Uso:
```rust
// Use o RateLimitLayer do Tower e converta-o para um formato compatível com o Salvo através do método compat()
let limit = RateLimitLayer::new(5, Duration::from_secs(30)).compat();
let router = Router::new().hoop(limit).get(hello);
```

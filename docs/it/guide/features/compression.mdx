# Compressione della Risposta

Un middleware per comprimere il contenuto della `Response`.

Supporta tre formati di compressione: `br`, `gzip` e `deflate`. La priorità e la configurazione di ciascun metodo di compressione possono essere personalizzate secondo necessità.

## Compressione dei Dati nel Protocollo HTTP

1. Livello formato file: tipi di file specifici utilizzano algoritmi di compressione dedicati.
2. Livello protocollo HTTP: la trasmissione compressa delle risorse end-to-end è ottenuta tramite la codifica del contenuto.
3. Livello connessione: la compressione dei dati viene eseguita tra i nodi di una connessione HTTP.

Per maggiori informazioni sulla compressione HTTP, consultare la [documentazione di Mozilla Developer Network](https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Compression).

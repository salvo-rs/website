# Catcher

import { Tab, Tabs } from '@rspress/core/theme';

Wenn der Statuscode der `Response` einen Fehler anzeigt und der `Body` auf der Seite leer ist, versucht Salvo, diesen Fehler mit einem `Catcher` abzufangen, um eine benutzerfreundliche Fehlerseite anzuzeigen.

Sie können mit `Catcher::default()` einen standardmäßigen `Catcher` des Systems zurückgeben und ihn dann zum `Service` hinzufügen.

```rust
use salvo::catcher::Catcher;

Service::new(router).catcher(Catcher::default());
```

Der Standard-`Catcher` unterstützt das Senden von Fehlerseiten in den Formaten `XML`, `JSON`, `HTML` und `Text`.

Sie können dem Standard-`Catcher` benutzerdefinierte Fehlerbehandlungsroutinen hinzufügen, indem Sie ihm einen `hoop` hinzufügen. Diese Fehlerbehandlungsroutinen sind weiterhin `Handler`.

Sie können dem `Catcher` über `hoop` mehrere benutzerdefinierte Fehlerbehandlungsroutinen hinzufügen. Benutzerdefinierte Fehlerbehandlungsroutinen können nach der Fehlerbehandlung die Methode `FlowCtrl::skip_next` aufrufen, um nachfolgende Fehlerbehandlungen zu überspringen und frühzeitig zurückzukehren.

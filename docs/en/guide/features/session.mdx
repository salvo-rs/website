Middleware that provides `Session` support.

## What is a Session

A Session is a piece of user data stored on the server-side, typically used to track user status and save temporary data. In web applications, Sessions solve the stateless nature of the HTTP protocol, allowing the server to recognize multiple requests from the same user.

## Using Sessions in Salvo

In Salvo, the Session middleware provides a simple API to read and write session data. You can obtain a session instance via `SessionDepot`, and manage session data using methods like `get`, `set`, and `remove`.

## Common Use Cases

- User authentication and login status management
- Temporary storage of form data
- Storage of user preferences

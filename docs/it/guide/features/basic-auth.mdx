# Autenticazione di Base

Middleware che fornisce supporto per l'autenticazione Basic Auth.

## Introduzione a Basic Auth

Basic Auth è un semplice meccanismo di autenticazione HTTP che trasmette nome utente e password tramite il campo `Authorization` nell'intestazione della richiesta HTTP. Il formato è `Authorization: Basic <base64(username:password)>`. Sebbene semplice, poiché le credenziali sono solo codificate in Base64 e non crittografate, di solito richiede l'uso con HTTPS per garantire la sicurezza.

## Confronto delle implementazioni di Basic Auth nei framework comuni

| Framework | Linguaggio | Implementazione Basic Auth |
|-----------|------------|----------------------------|
| Salvo | Rust | Implementato tramite middleware `BasicAuth` e `BasicAuthValidator` personalizzato |
| Express | JavaScript | Utilizza il pacchetto middleware `express-basic-auth` |
| Spring Security | Java | Configurato tramite `httpBasic()` e implementato con `UserDetailsService` |
| ASP.NET Core | C# | Utilizza `.UseAuthentication()` e `AddAuthentication(AuthenticationSchemes.Basic)` |
| Gin | Go | Tramite middleware personalizzato o utilizzando il pacchetto `gin-contrib/auth` |

## Casi d'uso

Basic Auth è adatto per i seguenti scenari:

1. **API e strumenti interni**: strumenti di gestione e API utilizzati internamente all'azienda
2. **Ambienti di sviluppo e test**: implementazione rapida dell'autenticazione senza bisogno di sistemi di login complessi
3. **Protezione semplice delle API**: quando non è necessario un sistema complesso di gestione utenti
4. **In combinazione con altre misure di sicurezza**: come parte di un'architettura di sicurezza multilivello

In Salvo, il middleware Basic Auth può essere facilmente integrato nelle route, personalizzando la logica di validazione implementando il trait `BasicAuthValidator`, offrendo grande flessibilità.

## Note importanti

- Utilizzare sempre con HTTPS per proteggere la trasmissione delle credenziali
- Non adatto per ambienti di produzione che memorizzano informazioni sensibili
- Considerare l'uso di metodi di autenticazione più sicuri come JWT, OAuth, ecc. per ambienti di produzione

# Authentification JWT

JWT (JSON Web Token) est un standard ouvert (RFC 7519) permettant de transmettre des informations de manière sécurisée entre différentes parties. Il s'agit d'une méthode compacte et sûre pour les URL, représentant des revendications sous forme d'objet JSON, généralement utilisées pour l'authentification et l'échange d'informations.

Un JWT se compose de trois parties :
- **En-tête (Header)** - Spécifie le type de jeton et l'algorithme de signature utilisé
- **Charge utile (Payload)** - Contient les revendications (claims), telles que l'ID utilisateur, les rôles, la date d'expiration, etc.
- **Signature (Signature)** - Utilisée pour vérifier que le message n'a pas été modifié pendant la transmission

Utilisation typique du JWT dans un processus d'authentification :
1. Après la connexion de l'utilisateur, le serveur génère un jeton JWT
2. Le jeton est renvoyé au client et stocké (généralement dans le localStorage ou un cookie)
3. Dans les requêtes suivantes, le client inclut ce jeton dans l'en-tête Authorization
4. Le serveur vérifie la validité du jeton et autorise l'accès

# WebTransport

WebTransport — это сетевой транспортный протокол на основе HTTP/3, обеспечивающий двустороннюю связь между клиентами и серверами с гарантией низкой задержки, высокой пропускной способности и безопасности.

## Сценарии использования

WebTransport особенно подходит для следующих случаев:

- **Приложения реального времени**: онлайн-игры, инструменты совместной работы, видеоконференции и другие приложения, требующие связи с минимальной задержкой.
- **Передача больших файлов**: поддерживает высокоскоростную передачу данных, идеально подходит для потоковой передачи медиа и загрузки/выгрузки крупных файлов.
- **Мультиплексированная связь**: позволяет одновременно устанавливать несколько двунаправленных и однонаправленных потоков данных.
- **Датаграммная связь**: поддерживает передачу датаграмм без гарантии порядка доставки и надёжности, что подходит для сценариев с экстремальными требованиями к реальному времени.

По сравнению с WebSocket, WebTransport обеспечивает меньшую задержку и более гибкие режимы связи, демонстрируя особые преимущества в нестабильных сетевых условиях.

## Реализация в Salvo

- Поддержку установки сессий WebTransport
- Поддержку двунаправленной потоковой связи
- Поддержку однонаправленной потоковой связи
- Поддержку передачи датаграмм
- Возможность инициирования потоков связи сервером

### Конфигурация и запуск

Для запуска приложения Salvo с поддержкой WebTransport требуется настройка TLS-сертификатов и QUIC-листенера:

```rust
let cert = include_bytes!("../certs/cert.pem").to_vec();
let key = include_bytes!("../certs/key.pem").to_vec();

// Конфигурация маршрутизации
let router = Router::new().push(Router::with_path("counter").goal(connect));

// Настройка TLS
let config = RustlsConfig::new(Keycert::new().cert(cert.as_slice()).key(key.as_slice()));

// Настройка листенеров
let listener = TcpListener::new(("0.0.0.0", 5800)).rustls(config.clone());
let acceptor = QuinnListener::new(config, ("0.0.0.0", 5800))
    .join(listener)
    .bind()
    .await;

// Запуск сервера
Server::new(acceptor).serve(router).await;
```

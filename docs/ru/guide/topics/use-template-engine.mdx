# Использование шаблонизаторов

import { Tab, Tabs } from '@rspress/core/theme';

Salvo не включает встроенные шаблонизаторы, поскольку выбор стиля шаблонизатора — дело личных предпочтений.

По своей сути шаблонизатор — это: данные + шаблон = строка.

Таким образом, любой шаблонизатор можно использовать, если он способен рендерить итоговую строку.

Например, поддержка `askama`:

<Tabs>
  <Tab label="main.rs">
  ```rust file="<root>/codes/template-askama/src/main.rs"
  ```
  </Tab>
  <Tab label="Cargo.toml">
  ```toml file="<root>/codes/template-askama/Cargo.toml"
  ```
  </Tab>
  <Tab label="templates/hello.html">
  ```html file="<root>/codes/template-askama/templates/hello.html"
  ```
  </Tab>
</Tabs>

**Примечание**: Для проектов без особой сложности мы рекомендуем использовать подход с разделением фронтенда и бэкенда. В этом случае фронтенд строится на более гибких и развитых UI-фреймворках (таких как React, Vue, Svelte и др.), а Salvo выступает в роли бэкенд-API. Такой подход повышает эффективность разработки, чётко разделяет обязанности между фронтендом и бэкендом и лучше соответствует современным тенденциям в создании веб-приложений.

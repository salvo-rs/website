# Compressão de Resposta

Middleware para processamento de compressão de conteúdo em `Response`.

Oferece suporte a três formatos de compressão: `br`, `gzip`, `deflate`. É possível configurar prioridades e outros parâmetros para cada método de compressão conforme necessário.

## Compressão de Dados no Protocolo HTTP

1. Nível de formato de arquivo: tipos específicos de arquivos utilizam algoritmos de compressão especializados
2. Nível de protocolo HTTP: compressão de recursos de ponta a ponta através de codificação de conteúdo
3. Nível de conexão: compressão de dados entre nós da conexão HTTP

Para mais informações sobre compressão HTTP, consulte a [documentação da Mozilla Developer Network](https://developer.mozilla.org/pt-BR/docs/Web/HTTP/Guides/Compression).

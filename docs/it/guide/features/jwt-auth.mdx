# Autenticazione JWT

JWT (JSON Web Token) è uno standard aperto (RFC 7519) utilizzato per trasmettere informazioni in modo sicuro tra le parti. È un metodo compatto e sicuro per URL che rappresenta dichiarazioni sotto forma di oggetto JSON, comunemente impiegato per l'autenticazione e lo scambio di informazioni.

Il JWT è composto da tre parti:
- **Intestazione (Header)** - Specifica il tipo di token e l'algoritmo di firma utilizzato
- **Carico utile (Payload)** - Contiene le dichiarazioni (claims), come ID utente, ruoli, data di scadenza, ecc.
- **Firma (Signature)** - Utilizzata per verificare che il messaggio non sia stato alterato durante la trasmissione

Utilizzo tipico del JWT nel flusso di autenticazione:
1. Dopo il login dell'utente, il server genera un token JWT
2. Il token viene restituito al client e memorizzato (solitamente in localStorage o cookie)
3. Nelle richieste successive, il client include il token nell'intestazione Authorization
4. Il server verifica la validità del token e autorizza l'accesso

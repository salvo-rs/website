# Reverse-Proxy

Ein Reverse-Proxy ist eine Serverarchitektur, die Anfragen von Clients entgegennimmt und an einen oder mehrere Backend-Server weiterleitet. Im Gegensatz zu einem Forward-Proxy (der für Clients agiert) arbeitet der Reverse-Proxy auf Serverseite.

Die Hauptvorteile eines Reverse-Proxy:
- Lastverteilung: Verteilung von Anfragen auf mehrere Server
- Erhöhte Sicherheit: Verbergen echter Serverinformationen
- Inhaltscaching: Leistungssteigerung
- Pfadumleitung und Weiterleitung: Flexible Anfragenrouting

Das Salvo-Framework bietet ein Middleware-Modul für Reverse-Proxy-Funktionalität.

_**Beispielcode**_ 

```rust
use salvo::prelude::*;
use salvo::proxy::Proxy;

#[tokio::main]
async fn main() {
    tracing_subscriber::fmt().init();
    
    let router = Router::new()
        .push(
            Router::new()
                .path("google/{**rest}")
                .handle(Proxy::<Vec<&str>>::new(vec!["https://www.google.com"])),
        )
        .push(
            Router::new()
                .path("baidu/{**rest}")
                .handle(Proxy::<Vec<&str>>::new(vec!["https://www.baidu.com"])),
        );
    let acceptor = TcpListener::new("127.0.0.1:5800").bind().await; Server::new(acceptor).serve(router).await;
}
```

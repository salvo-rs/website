# Compatibilité avec les Middlewares Tower

import { Tab, Tabs } from '@rspress/core/theme';

Salvo offre une prise en charge de la compatibilité avec l'écosystème Tower via la fonctionnalité `tower-compat`. Pour les API spécifiques, veuillez [consulter la documentation](https://docs.rs/salvo_extra/latest/salvo_extra/tower_compat/index.html).

## Concepts Clés de Tower

Tower est une bibliothèque d'abstraction de services, principalement basée sur deux traits fondamentaux :

### Le Trait Service

```rust
pub trait Service<Request> {
    type Response;
    type Error;
    type Future: Future<Output = Result<Self::Response, Self::Error>>;

    fn poll_ready(&mut self, cx: &mut Context<'_>) -> Poll<Result<(), Self::Error>>;
    fn call(&mut self, req: Request) -> Self::Future;
}
```

Le `Service` est responsable du traitement des requêtes et du renvoi des réponses, similaire au gestionnaire (Handler) de Salvo.

### Le Trait Layer

```rust
pub trait Layer<S> {
    type Service;
    fn layer(&self, inner: S) -> Self::Service;
}
```

Le `Layer` est utilisé pour encapsuler un service et ajouter des fonctionnalités supplémentaires, similaire aux middlewares de Salvo.

## Prise en Charge de la Compatibilité par Salvo

Salvo fournit deux implémentations de traits clés pour assurer la compatibilité avec l'écosystème Tower :

- **TowerLayerCompat** : Convertit un `Layer` de Tower en `Handler` de Salvo, permettant son utilisation en tant que Hoop (middleware).
- **TowerServiceCompat** : Convertit un `Service` de Tower en `Handler` de Salvo.

Exemple d'utilisation :
```rust
// Utilisez RateLimitLayer de Tower et convertissez-le au format compatible Salvo via la méthode compat()
let limit = RateLimitLayer::new(5, Duration::from_secs(30)).compat();
let router = Router::new().hoop(limit).get(hello);
```

# JWT認証

JWT（JSON Web Token）は、RFC 7519で定義されたオープンスタンダードで、関係者間で安全に情報を伝達するための仕様です。JSONオブジェクト形式でクレームを表現するコンパクトでURLセーフな方法であり、主に認証や情報交換に使用されます。

JWTは以下の3つの部分で構成されます：
- **ヘッダー(Header)** - トークンのタイプと使用する署名アルゴリズムを指定
- **ペイロード(Payload)** - ユーザーID、ロール、有効期限などのクレームを含む
- **署名(Signature)** - メッセージが改ざんされていないことを検証するための部分

認証フローにおけるJWTの典型的な使用方法：
1. ユーザーがログインすると、サーバーがJWTトークンを生成
2. トークンがクライアントに返され（通常はlocalStorageやcookieに）保存
3. 以降のリクエストでは、クライアントがAuthorizationヘッダーにこのトークンを付与
4. サーバーがトークンの有効性を検証し、アクセスを許可

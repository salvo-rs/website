# Reqwest: Biblioteca Cliente HTTP em Rust

[Reqwest](https://docs.rs/reqwest/latest/reqwest/) é uma biblioteca cliente HTTP de alto nível que simplifica o processo de manipulação de requisições HTTP, oferecendo diversas funcionalidades comuns:

- Suporte a APIs assíncronas e bloqueantes
- Manipulação de diversos tipos de corpo de requisição: texto simples, JSON, formulários codificados em URL, formulários multiparte
- Política de redirecionamento personalizável
- Suporte a proxy HTTP
- Uso padrão de criptografia TLS
- Gerenciamento de cookies

## Uso Básico

### Realizando uma Requisição GET

Para uma única requisição, pode-se usar o método abreviado `get`:

```rust
let body = reqwest::get("https://www.rust-lang.org")
    .await?
    .text()
    .await?;

println!("body = {body:?}");
```

> Nota: Se planeja executar múltiplas requisições, é melhor criar um `Client` e reutilizá-lo para aproveitar os benefícios do pool de conexões.

```rust
let client = reqwest::Client::new();
let res = client.get("https://www.rust-lang.org")
    .send()
    .await?;
```

### Realizando uma Requisição POST

Use o método `body()` para definir o corpo da requisição:

```rust
let client = reqwest::Client::new();
let res = client.post("http://httpbin.org/post")
    .body("o corpo exato a ser enviado")
    .send()
    .await?;
```

### Dados de Formulário

Enviar dados de formulário é uma necessidade comum. Pode-se usar qualquer tipo que possa ser serializado em dados de formulário:

```rust
// Isso enviará uma requisição POST com o corpo `foo=bar&baz=quux`
let params = [("foo", "bar"), ("baz", "quux")];
let client = reqwest::Client::new();
let res = client.post("http://httpbin.org/post")
    .form(&params)
    .send()
    .await?;
```

### Dados JSON

Enviar dados JSON é fácil com o método `json` (requer o recurso `json`):

```rust
// Isso enviará uma requisição POST com o corpo `{"lang":"rust","body":"json"}`
let mut map = HashMap::new();
map.insert("lang", "rust");
map.insert("body", "json");

let client = reqwest::Client::new();
let res = client.post("http://httpbin.org/post")
    .json(&map)
    .send()
    .await?;
```

### Manipulação da Resposta

A resposta pode ser manipulada de várias maneiras:

```rust
let res = client.get("https://www.rust-lang.org").send().await?;

// Obter o código de status
let status = res.status();

// Obter cabeçalhos da resposta
let content_type = res.headers().get("content-type").unwrap();

// Ler o corpo da resposta como texto
let body = res.text().await?;

// Ou analisar como JSON
let json: serde_json::Value = res.json().await?;
```

## Funcionalidades Avançadas

### Política de Redirecionamento

Por padrão, o cliente lida automaticamente com redirecionamentos HTTP, seguindo até 10 redirecionamentos. Esse comportamento pode ser personalizado usando `ClientBuilder`:

```rust
let custom_client = reqwest::Client::builder()
    .redirect(reqwest::redirect::Policy::none()) // Desabilita redirecionamentos
    .build()?;
```

### Suporte a Cookies

O armazenamento e envio automático de cookies de sessão pode ser habilitado via `ClientBuilder`:

```rust
let client = reqwest::Client::builder()
    .cookie_store(true)
    .build()?;
```

### Configuração de Proxy

O proxy do sistema é habilitado por padrão, procurando configurações de proxy HTTP ou HTTPS em variáveis de ambiente:

- `HTTP_PROXY` ou `http_proxy`: proxy para conexões HTTP
- `HTTPS_PROXY` ou `https_proxy`: proxy para conexões HTTPS
- `ALL_PROXY` ou `all_proxy`: proxy para ambos os tipos de conexão

Também é possível definir um proxy explicitamente via código:

```rust
let proxy = reqwest::Proxy::http("https://secure.example")?;
let client = reqwest::Client::builder()
    .proxy(proxy)
    .build()?;

// Ou desabilitar proxies
let client = reqwest::Client::builder()
    .no_proxy()
    .build()?;
```

### Configuração TLS

O cliente usa TLS por padrão para se conectar a destinos HTTPS:

```rust
// Adicionar um certificado de servidor adicional
let cert = reqwest::Certificate::from_pem(&cert_bytes)?;
let client = reqwest::Client::builder()
    .add_root_certificate(cert)
    .build()?;

// Configurar um certificado de cliente
let identity = reqwest::Identity::from_pkcs12_der(&pkcs12_der, "password")?;
let client = reqwest::Client::builder()
    .identity(identity)
    .build()?;
```

### Configuração de Timeout

Timeouts podem ser configurados para as requisições:

```rust
let client = reqwest::Client::builder()
    .timeout(std::time::Duration::from_secs(10))
    .build()?;
```

## Recursos Opcionais

O Reqwest oferece vários recursos opcionais que podem ser habilitados ou desabilitados via recursos do Cargo:

- `http2` (habilitado por padrão): suporte a HTTP/2
- `default-tls` (habilitado por padrão): fornece suporte TLS para HTTPS
- `rustls-tls`: fornece funcionalidade TLS usando rustls
- `blocking`: fornece uma API de cliente bloqueante
- `json`: fornece serialização e desserialização JSON
- `multipart`: fornece funcionalidade de formulário multiparte
- `cookies`: fornece suporte a sessão de cookies
- `gzip`, `brotli`, `deflate`, `zstd`: suporte a várias descompressões de corpo de resposta
- `socks`: fornece suporte a proxy SOCKS5

## API Bloqueante

Quando a operação assíncrona não é necessária, pode-se usar a API bloqueante (requer o recurso `blocking`):

```rust
let body = reqwest::blocking::get("https://www.rust-lang.org")?.text()?;

let client = reqwest::blocking::Client::new();
let res = client.post("http://httpbin.org/post")
    .json(&map)
    .send()?;
```

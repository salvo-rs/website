# Ограничение частоты запросов

Промежуточное ПО, обеспечивающее функциональность ограничения частоты запросов.

import { Tab, Tabs } from '@rspress/core/theme';

## Ключевые особенности

* `RateIssuer` предоставляет абстракцию для идентификации ключей посетителей. `RemoteIpIssuer` является одной из его реализаций, определяющей посетителей на основе их IP-адресов. Ключи не обязательно должны быть строками; любой тип, удовлетворяющий ограничениям `Hash + Eq + Send + Sync + 'static`, может служить ключом.

* `RateGuard` предлагает абстракцию для алгоритмов ограничения частоты запросов. Стандартные реализации включают фиксированное окно (`FixedGuard`) и скользящее окно (`SlidingGuard`).

* `RateStore` предоставляет операции хранения данных. `MokaStore` — это встроенная реализация кэша в памяти на основе `moka`. Вы также можете определить собственную реализацию.

* `RateLimiter` — это структура, реализующая `Handler`, которая включает поле `skipper` для указания запросов, которые должны обходить ограничение частоты. По умолчанию используется `none_skipper`, что означает, что никакие запросы не пропускаются.

* `QuotaGetter` предоставляет абстракцию для получения квот. Он извлекает объект квоты на основе `Key` посетителя, позволяя хранить пользовательские квоты и другие конфигурации в базах данных, а также динамически изменять или получать их.

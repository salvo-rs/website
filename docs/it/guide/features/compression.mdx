# Compressione della risposta

Middleware per la compressione del contenuto della `Response`.

Fornisce supporto per tre formati di compressione: `br`, `gzip`, `deflate`. È possibile configurare la priorità e altri parametri per ciascun metodo di compressione in base alle esigenze.

## Compressione dei dati nel protocollo HTTP

1. Livello formato file: algoritmi di compressione specifici per determinati tipi di file
2. Livello protocollo HTTP: compressione end-to-end delle risorse tramite codifica del contenuto
3. Livello connessione: compressione dei dati tra i nodi della connessione HTTP

Per maggiori informazioni sulla compressione HTTP, consultare la [documentazione di Mozilla Developer Network](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Guides/Compressione).

# Estado Afijo Compartir datos en solicitudes

import { Tab, Tabs } from '@rspress/core/theme';

El middleware Estado Afijo se utiliza para agregar datos compartidos al Depot.
Para utilizar la funcionalidad Estado Afijo, es necesario habilitar la feature `affix-state` en Cargo.toml.

## Análisis de funcionalidad
Estado Afijo proporciona una forma sencilla de compartir datos durante el procesamiento de solicitudes. Te permite:

Inyectar configuraciones globales o datos compartidos durante la fase de configuración de rutas
Acceder a estos datos a través del Depot en cualquier manejador
Soporta cualquier tipo clonable como datos de estado

## Comparación con otros marcos para una comprensión rápida del concepto
| Marco | Lenguaje | Método de gestión de estado |
|------|------|-------------|
| Salvo (Estado Afijo) | Rust | Almacena y accede a través del Depot, soporta múltiples tipos |
| Axum | Rust | Almacena estado a través de Extension, similar pero con diferente uso |
| Actix-web | Rust | Utiliza App Data y Web::Data para compartir estado |
| Gin | Go | Usa context.Set y context.Get para almacenar y recuperar datos |
| Echo | Go | Utiliza context.Set y context.Get para gestionar estado compartido |
| Spring | Java | Usa ApplicationContext o anotación @Bean para gestionar dependencias |
| Quarkus | Java | Utiliza CDI y mecanismos de inyección de dependencias |
| Express.js | JavaScript | Usa app.locals o req.app.locals para almacenar estado global |
| Nest.js | JavaScript | Utiliza sistema de inyección de dependencias para gestionar servicios compartidos |
| Koa.js | JavaScript | Usa ctx.state para almacenar estado a nivel de solicitud |

## Escenarios de uso comunes

- Compartir pool de conexiones a base de datos
- Compartir configuración de aplicación
- Compartir instancias de caché
- Compartir cliente de API
- Seguimiento de contadores globales o estado

La ventaja de Estado Afijo radica en su simplicidad y flexibilidad, permitiendo compartir fácilmente cualquier tipo de datos entre diferentes rutas y manejadores sin necesidad de código repetitivo extenso.

# Compressão de Resposta

Um middleware para comprimir o conteúdo de `Response`.

Suporta três formatos de compressão: `br`, `gzip` e `deflate`. A prioridade e configuração de cada método de compressão podem ser personalizadas conforme necessário.

## Compressão de Dados no Protocolo HTTP

1. Nível de formato de arquivo: Tipos de arquivo específicos utilizam algoritmos de compressão dedicados.
2. Nível de protocolo HTTP: Transmissão de recursos com compressão ponta a ponta é alcançada através da codificação de conteúdo.
3. Nível de conexão: A compressão de dados é realizada entre os nós de uma conexão HTTP.

Para mais informações sobre compressão HTTP, consulte a [documentação da Mozilla Developer Network](https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Compression).

# Komprimierung von Antworten

Middleware zur Komprimierung von `Response`-Inhalten.

Bietet Unterstützung für drei Komprimierungsformate: `br`, `gzip`, `deflate`. Prioritäten und andere Einstellungen der Komprimierungsmethoden können nach Bedarf konfiguriert werden.

## Datenkomprimierung im HTTP-Protokoll

1. Auf Dateiformatebene: Spezifische Dateitypen verwenden dedizierte Komprimierungsalgorithmen
2. Auf HTTP-Protokollebene: End-to-End-Ressourcenkomprimierung durch Content-Encoding
3. Auf Verbindungsebene: Datenkomprimierung zwischen den Knotenpunkten einer HTTP-Verbindung

Weitere Informationen zur HTTP-Komprimierung finden Sie in der [Dokumentation des Mozilla Developer Network](https://developer.mozilla.org/de/docs/Web/HTTP/Guides/Compression).

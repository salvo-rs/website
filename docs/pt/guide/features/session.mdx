Middleware que fornece suporte para `Session`.

## O que é Sessão

Sessão é um conjunto de dados do usuário armazenados no lado do servidor, geralmente usados para rastrear o estado do usuário e salvar dados temporários. Em aplicações web, a Sessão pode resolver o problema de ausência de estado no protocolo HTTP, permitindo que o servidor identifique múltiplas requisições do mesmo usuário.

## Uso de Sessão no Salvo

No Salvo, o middleware de Sessão fornece uma API simples para ler e escrever dados de sessão. Através do `SessionDepot` é possível obter a instância da sessão, e com métodos como `get`, `set`, `remove` entre outros, é possível gerenciar os dados da sessão.

## Usos comuns

- Autenticação de usuário e gerenciamento de estado de login
- Armazenamento temporário de dados de formulários
- Armazenamento de preferências do usuário

# Autenticação JWT

JWT (JSON Web Token) é um padrão aberto (RFC 7519) usado para transmitir informações de forma segura entre partes. É uma maneira compacta e segura para URLs de representar declarações como objetos JSON, que são normalmente usados para autenticação e troca de informações.

O JWT consiste em três partes:
- **Cabeçalho (Header)** - Especifica o tipo do token e o algoritmo de assinatura utilizado
- **Carga útil (Payload)** - Contém as declarações (claims), como ID do usuário, funções, tempo de expiração, etc.
- **Assinatura (Signature)** - Usada para verificar se a mensagem não foi alterada durante a transmissão

O uso típico do JWT no fluxo de autenticação:
1. Após o login do usuário, o servidor gera um token JWT
2. O token é retornado ao cliente e armazenado (geralmente em localStorage ou cookie)
3. Nas requisições subsequentes, o cliente envia este token no cabeçalho Authorization
4. O servidor verifica a validade do token e autoriza o acesso

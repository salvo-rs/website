# Catcher

import { Tab, Tabs } from '@rspress/core/theme';

Si el código de estado de la `Response` es un error y el `Body` de la página está vacío, Salvo intentará capturar este error utilizando un `Catcher` para mostrar una página de error amigable.

Puedes obtener un `Catcher` predeterminado del sistema mediante `Catcher::default()` y luego agregarlo al `Service`.

```rust
use salvo::catcher::Catcher;

Service::new(router).catcher(Catcher::default());
```

El `Catcher` predeterminado admite el envío de páginas de error en formatos `XML`, `JSON`, `HTML` y `Text`.

Puedes agregar un programa de captura de errores personalizado al `Catcher` añadiendo un `hoop` a este `Catcher` predeterminado. Este programa de captura de errores sigue siendo un `Handler`.

Puedes agregar múltiples programas de captura de errores personalizados al `Catcher` mediante `hoop`. Los manejadores de errores personalizados pueden llamar al método `FlowCtrl::skip_next` después de procesar el error para omitir los programas de error posteriores y retornar anticipadamente.

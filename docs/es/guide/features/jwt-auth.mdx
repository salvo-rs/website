# Autenticación JWT

JWT (JSON Web Token) es un estándar abierto (RFC 7519) utilizado para transmitir información de forma segura entre partes. Es un método compacto y seguro para URL que representa declaraciones en forma de objeto JSON, comúnmente empleado para autenticación e intercambio de información.

Un JWT consta de tres partes:
- **Encabezado (Header)** - Especifica el tipo de token y el algoritmo de firma utilizado
- **Carga útil (Payload)** - Contiene las declaraciones (claims), como ID de usuario, roles, tiempo de expiración, etc.
- **Firma (Signature)** - Se utiliza para verificar que el mensaje no ha sido alterado durante la transmisión

Uso típico de JWT en flujos de autenticación:
1. Tras el inicio de sesión del usuario, el servidor genera un token JWT
2. El token se devuelve al cliente y se almacena (normalmente en localStorage o cookies)
3. En solicitudes posteriores, el cliente incluye este token en la cabecera Authorization
4. El servidor verifica la validez del token y autoriza el acceso

Proporciona middleware para la autenticación mediante JWT Auth.

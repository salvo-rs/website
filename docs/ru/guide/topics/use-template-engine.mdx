# Использование шаблонизаторов

import { Tab, Tabs } from '@rspress/core/theme';

Salvo не включает встроенные шаблонизаторы, поскольку предпочтения в стилях шаблонизаторов у всех разные.

По своей сути, шаблонизатор — это просто: данные + шаблон = строка.

Таким образом, любой шаблонизатор может быть поддержан, если конечная строка может быть отрендерена.

Например, поддержка `askama`:

<Tabs>
  <Tab label="main.rs">
  ```rust file="<root>/codes/template-askama/src/main.rs"
  ```
  </Tab>
  <Tab label="Cargo.toml">
  ```toml file="<root>/codes/template-askama/Cargo.toml"
  ```
  </Tab>
  <Tab label="templates/hello.html">
  ```html file="<root>/codes/template-askama/templates/hello.html"
  ```
  </Tab>
</Tabs>

**Примечание**: Для проектов, которые не являются особенно сложными, мы настоятельно рекомендуем использовать подход разделения фронтенда и бэкенда. Используйте более гибкие и богатые экосистемой UI-фреймворки (такие как React, Vue, Svelte и т.д.) для построения фронтенда, а Salvo будет служить бэкенд-API сервисом. Этот подход приводит к более высокой эффективности разработки, более четкому разделению обязанностей между фронтендом и бэкендом и лучше соответствует современным тенденциям разработки веб-приложений.

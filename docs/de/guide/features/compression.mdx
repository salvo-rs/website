# Antwortkomprimierung

Ein Middleware zur Komprimierung des Inhalts von `Response`.

Unterstützt drei Komprimierungsformate: `br`, `gzip` und `deflate`. Die Priorität und Konfiguration jeder Komprimierungsmethode kann nach Bedarf angepasst werden.

## Datenkomprimierung im HTTP-Protokoll

1. Dateiformatebene: Bestimmte Dateitypen verwenden spezielle Komprimierungsalgorithmen.
2. HTTP-Protokollebene: Ende-zu-Ende-Ressourcenkomprimierung wird durch Content-Encoding erreicht.
3. Verbindungsebene: Datenkomprimierung erfolgt zwischen den Knoten einer HTTP-Verbindung.

Weitere Informationen zur HTTP-Komprimierung finden Sie in der [Mozilla Developer Network-Dokumentation](https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/Compression).

# Encabezados de Caché

Middleware que proporciona soporte para la configuración de encabezados de caché.

El control de caché es una parte fundamental de la optimización del rendimiento web. Al configurar correctamente los encabezados de caché, se pueden reducir solicitudes de red innecesarias y mejorar el rendimiento de la aplicación. [Cache-Control](https://developer.mozilla.org/es/docs/Web/HTTP/Headers/Cache-Control) es una directiva en los encabezados de respuesta HTTP que especifica la política de caché del navegador, controlando quién puede almacenar en caché la respuesta, bajo qué condiciones y por cuánto tiempo.

La implementación interna incluye tres manejadores (`Handler`): `CachingHeaders`, `Modified` y `ETag`. `CachingHeaders` es una combinación de los dos últimos. En condiciones normales, se recomienda utilizar `CachingHeaders`.

- `Modified`: Proporciona validación de caché basada en la última fecha de modificación del recurso.
- `ETag`: Utiliza etiquetas de entidad (Entity Tag) para ofrecer un mecanismo de validación de recursos más preciso.
- `CachingHeaders`: Combina ambos mecanismos anteriores para brindar un soporte completo de control de caché.

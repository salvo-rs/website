Промежуточное ПО, обеспечивающее поддержку `Session`.

## Что такое Session

Session — это данные пользователя, хранящиеся на стороне сервера, обычно используемые для отслеживания состояния пользователя и сохранения временных данных. В веб-приложениях Session решает проблему отсутствия состояния в протоколе HTTP, позволяя серверу идентифицировать множественные запросы от одного и того же пользователя.

## Использование Session в Salvo

В Salvo промежуточное ПО Session предоставляет простой API для чтения и записи данных сессии. Через `SessionDepot` можно получить экземпляр сессии, а с помощью методов `get`, `set`, `remove` и других — управлять данными сессии.

## Типичные сценарии использования

- Аутентификация пользователя и управление статусом входа
- Временное хранение данных форм
- Сохранение пользовательских настроек

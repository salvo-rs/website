# Compatibilidad con Middleware Tower

import { Tab, Tabs } from '@rspress/core/theme';

Salvo proporciona soporte de compatibilidad con el ecosistema Tower a través de la funcionalidad `tower-compat`. Para conocer las API específicas, [consulta la documentación](https://docs.rs/salvo_extra/latest/salvo_extra/tower_compat/index.html).

## Conceptos Clave de Tower

Tower es una biblioteca de abstracción de servicios basada principalmente en dos traits fundamentales:

### Trait Service

```rust
pub trait Service<Request> {
    type Response;
    type Error;
    type Future: Future<Output = Result<Self::Response, Self::Error>>;

    fn poll_ready(&mut self, cx: &mut Context<'_>) -> Poll<Result<(), Self::Error>>;
    fn call(&mut self, req: Request) -> Self::Future;
}
```

El `Service` se encarga de procesar solicitudes y devolver respuestas, similar a los manejadores (handlers) de Salvo.

### Trait Layer

```rust
pub trait Layer<S> {
    type Service;
    fn layer(&self, inner: S) -> Self::Service;
}
```

El `Layer` se utiliza para envolver servicios y agregar funcionalidad adicional, similar al middleware de Salvo.

## Soporte de Compatibilidad de Salvo

Salvo proporciona la implementación de dos traits clave para lograr compatibilidad con el ecosistema Tower:

- **TowerLayerCompat**: Convierte un `Layer` de Tower en un `Handler` de Salvo, permitiendo su uso como Hoop (middleware).
- **TowerServiceCompat**: Convierte un `Service` de Tower en un `Handler` de Salvo.

Ejemplo de uso:
```rust
// Usa RateLimitLayer de Tower y conviértelo a un formato compatible con Salvo mediante el método compat()
let limit = RateLimitLayer::new(5, Duration::from_secs(30)).compat();
let router = Router::new().hoop(limit).get(hello);
```

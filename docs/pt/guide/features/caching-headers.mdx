# Cabeçalhos de Cache

Middleware que fornece suporte à configuração de cabeçalhos de cache.

O controle de cache é uma parte crucial da otimização de desempenho web. Ao configurar corretamente os cabeçalhos de cache, é possível reduzir solicitações de rede desnecessárias e melhorar o desempenho do aplicativo. [Cache-Control](https://developer.mozilla.org/pt-BR/docs/Web/HTTP/Headers/Cache-Control) é um cabeçalho de resposta HTTP usado para especificar diretivas de política de cache do navegador, controlando quem pode armazenar em cache a resposta, sob quais condições e por quanto tempo.

A implementação interna inclui três `Handlers`: `CachingHeaders`, `Modified` e `ETag`. `CachingHeaders` é uma combinação dos dois últimos. Normalmente, utiliza-se `CachingHeaders`.

- `Modified`: Fornece validação de cache com base na última data de modificação do recurso.
- `ETag`: Utiliza uma tag de entidade (Entity Tag) para fornecer um mecanismo de validação de recursos mais preciso.
- `CachingHeaders`: Combina os dois mecanismos acima, oferecendo suporte completo ao controle de cache.

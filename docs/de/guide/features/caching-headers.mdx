# Cache-Header

Middleware zur Unterstützung der Konfiguration von Cache-Headern.

Die Cache-Steuerung ist ein wesentlicher Bestandteil der Web-Performance-Optimierung. Durch korrektes Setzen von Cache-Headern können unnötige Netzwerkanfragen reduziert und die Anwendungsleistung verbessert werden. [Cache-Control](https://developer.mozilla.org/de/docs/Web/HTTP/Headers/Cache-Control) ist ein HTTP-Antwort-Header, der Anweisungen zur Festlegung der Browser-Cache-Strategie enthält. Er steuert, wer unter welchen Bedingungen Antworten cachen darf und wie lange diese zwischengespeichert werden.

Die Implementierung umfasst intern drei `Handler`: `CachingHeaders`, `Modified` und `ETag`. `CachingHeaders` ist eine Kombination der beiden letzteren. Normalerweise wird `CachingHeaders` verwendet.

- `Modified`: Bietet Cache-Validierung basierend auf der letzten Änderungszeit einer Ressource.
- `ETag`: Verwendet Entity Tags für einen präziseren Ressourcen-Validierungsmechanismus.
- `CachingHeaders`: Kombiniert die beiden oben genannten Mechanismen und bietet umfassende Cache-Steuerungsunterstützung.

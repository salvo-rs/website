# Intestazioni di Cache

Middleware che fornisce supporto per la configurazione delle intestazioni di cache.

Il controllo della cache è una parte cruciale dell'ottimizzazione delle prestazioni web. Impostando correttamente le intestazioni di cache, è possibile ridurre richieste di rete non necessarie e migliorare le prestazioni dell'applicazione. [Cache-Control](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Reference/Headers/Cache-Control) è un'intestazione di risposta HTTP che specifica le direttive per la strategia di caching del browser, controllando chi può memorizzare la risposta in cache, sotto quali condizioni e per quanto tempo.

L'implementazione interna include tre `Handler`: `CachingHeaders`, `Modified` e `ETag`. `CachingHeaders` è una combinazione degli altri due. Normalmente si utilizza `CachingHeaders`.

- `Modified`: Fornisce la validazione della cache basata sull'ultima modifica della risorsa.
- `ETag`: Utilizza l'Entity Tag per un meccanismo di validazione delle risorse più preciso.
- `CachingHeaders`: Combina i due meccanismi precedenti, offrendo un supporto completo per il controllo della cache.

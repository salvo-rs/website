# Arrêt gracieux

L'arrêt gracieux consiste à ne pas interrompre immédiatement toutes les connexions lors de la fermeture d'un serveur, mais plutôt à cesser d'accepter de nouvelles requêtes tout en laissant suffisamment de temps aux requêtes déjà acceptées pour terminer leur traitement avant d'arrêter le service. Cette approche évite l'interruption brutale des requêtes, améliorant ainsi l'expérience utilisateur et la fiabilité du système.

Salvo prend en charge l'arrêt gracieux via la méthode `handle` du `Server`, qui permet d'obtenir un gestionnaire de serveur, puis en appelant la méthode `stop_graceful`. Après l'appel de cette méthode, le serveur :

- Cesse d'accepter de nouvelles connexions
- Attend que les requêtes en cours soient traitées
- Force la fermeture des connexions restantes après un délai d'attente spécifié (si fourni)

Voici un exemple simple :

```rust
use salvo_core::prelude::*;

#[tokio::main]
async fn main() {
    let acceptor = TcpListener::new("127.0.0.1:5800").bind().await;
    let server = Server::new(acceptor);
    let handle = server.handle();

    // Arrêt gracieux du serveur
    tokio::spawn(async move {
        tokio::time::sleep(std::time::Duration::from_secs(60)).await;
        handle.stop_graceful(None);
    });
    server.serve(Router::new()).await;
}
```

Dans l'exemple ci-dessus :

- `server.handle()` obtient un gestionnaire du serveur, qui peut être utilisé pour contrôler son cycle de vie
- `handle.stop_graceful(None)` initie le processus d'arrêt gracieux, où `None` signifie qu'aucun délai d'attente n'est défini, attendant ainsi que toutes les requêtes soient traitées
- Pour définir un délai d'attente, vous pouvez passer `Some(Duration)`, après quoi les connexions restantes seront fermées de force

Cette approche est particulièrement adaptée aux applications déployées dans des environnements conteneurisés ou sur des plateformes cloud, ainsi qu'aux mises à jour à chaud pour garantir que les requêtes ne soient pas interrompues de manière inattendue.

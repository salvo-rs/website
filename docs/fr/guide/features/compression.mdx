# Compression des réponses

Middleware de traitement de compression pour le contenu des `Response`.

Prend en charge trois formats de compression : `br`, `gzip`, `deflate`. Permet de configurer la priorité de chaque méthode de compression selon les besoins.

## Compression de données dans le protocole HTTP

1. Au niveau du format de fichier : des algorithmes de compression spécifiques sont utilisés pour certains types de fichiers.
2. Au niveau du protocole HTTP : compression des ressources de bout en bout via l'encodage du contenu.
3. Au niveau de la connexion : compression des données entre les nœuds de connexion HTTP.

Pour plus d'informations sur la compression HTTP, consultez la [documentation du Mozilla Developer Network](https://developer.mozilla.org/fr/docs/Web/HTTP/Guides/Compression).

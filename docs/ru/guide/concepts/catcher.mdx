# Перехватчик

import { Tab, Tabs } from '@rspress/core/theme';

Когда `Response` возвращает код ошибки, а `Body` на странице пуст, Salvo попытается перехватить эту ошибку с помощью `Catcher` и отобразить удобную для пользователя страницу с ошибкой.

Вы можете получить системный `Catcher` по умолчанию, вызвав `Catcher::default()`, а затем добавить его в `Service`.

```rust
use salvo::catcher::Catcher;

Service::new(router).catcher(Catcher::default());
```

Стандартный `Catcher` поддерживает отправку страниц ошибок в форматах `XML`, `JSON`, `HTML` и `Text`.

Вы можете добавить пользовательские обработчики перехвата ошибок в `Catcher`, прикрепив `hoop` к этому стандартному `Catcher`. Эти обработчики перехвата ошибок по-прежнему имеют тип `Handler`.

Вы можете добавить несколько пользовательских обработчиков перехвата ошибок в `Catcher` через `hoop`. Пользовательские обработчики ошибок могут вызывать метод `FlowCtrl::skip_next` после обработки ошибки, чтобы пропустить последующие обработчики ошибок и завершить выполнение досрочно.

# Заголовки кэширования

Промежуточное ПО, обеспечивающее поддержку настройки заголовков кэширования.

Управление кэшем является важной частью оптимизации веб-производительности. Правильная настройка заголовков кэширования позволяет сократить ненужные сетевые запросы, тем самым повышая производительность приложения. [Cache-Control](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Cache-Control) — это заголовок HTTP-ответа, используемый для определения политик кэширования в браузере. Он контролирует, кто может кэшировать ответы, при каких условиях и как долго.

Внутренне он включает реализации трех обработчиков `Handler`: `CachingHeaders`, `Modified` и `ETag`. `CachingHeaders` представляет собой комбинацию двух последних. В обычных условиях используется `CachingHeaders`.

- `Modified`: Обеспечивает проверку кэша на основе времени последнего изменения ресурса.
- `ETag`: Использует Entity Tags для предоставления более точного механизма проверки ресурсов.
- `CachingHeaders`: Объединяет два вышеуказанных механизма для обеспечения комплексной поддержки управления кэшем.

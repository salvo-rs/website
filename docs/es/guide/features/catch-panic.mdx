# Capturar Pánico en las Solicitudes

Catch Panic se utiliza para capturar bloqueos que ocurren durante el procesamiento de solicitudes mientras el programa se ejecuta. Para conocer la API específica, [consulta la documentación](https://docs.rs/salvo_extra/latest/salvo_extra/catch_panic/index.html).

> **Nota**: Para usar `CatchPanic`, debes habilitar la característica `catch-panic` en `Cargo.toml`:
> ```toml
> salvo= { version = "xxx", features = ["catch-panic"] }
> ```

## Introducción al Middleware

`CatchPanic` es un middleware diseñado para capturar pánicos en los manejadores. Cuando ocurre un pánico durante el procesamiento de una solicitud, este middleware lo captura y escribe un error 500 Internal Server Error en la respuesta, evitando que el servidor completo colapse.

**Importante**: Este middleware debe utilizarse como el primer middleware para garantizar que pueda capturar pánicos provenientes de otros middlewares o manejadores.

## Uso Básico

```rust
use salvo_core::prelude::*;
use salvo_extra::catch_panic::CatchPanic;

#[handler]
async fn hello() {
    panic!("panic error!");
}

#[tokio::main]
async fn main() {
    let router = Router::new().hoop(CatchPanic::new()).get(hello);
    let acceptor = TcpListener::new("0.0.0.0:5800").bind().await;
    Server::new(acceptor).serve(router).await;
}
```

## Comparación con Otros Frameworks para una Comprensión Rápida

### Axum

Similar al middleware `catch_panic` de `Tower` en Axum:

```rust
use axum::{
    Router,
    routing::get,
    http::StatusCode,
};
use tower::ServiceBuilder;
use tower_http::catch_panic::CatchPanicLayer;

async fn panic_handler() -> &'static str {
    panic!("panic error!");
}

#[tokio::main]
async fn main() {
    let app = Router::new()
        .route("/", get(panic_handler))
        .layer(
            ServiceBuilder::new()
                .layer(CatchPanicLayer::new())
        );
    
    axum::Server::bind(&"0.0.0.0:3000".parse().unwrap())
        .serve(app.into_make_service())
        .await
        .unwrap();
}
```

import (

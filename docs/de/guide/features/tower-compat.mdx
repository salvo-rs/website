# Tower-Middleware-Kompatibilität

import { Tab, Tabs } from '@rspress/core/theme';

Salvo bietet Kompatibilitätsunterstützung für das Tower-Ökosystem über das Feature `tower-compat`. Für spezifische APIs [siehe bitte die Dokumentation](https://docs.rs/salvo_extra/latest/salvo_extra/tower_compat/index.html).

## Kernkonzepte von Tower

Tower ist eine Dienst-Abstraktionsbibliothek, die hauptsächlich auf zwei Kerneigenschaften basiert:

### Service-Trait

```rust
pub trait Service<Request> {
    type Response;
    type Error;
    type Future: Future<Output = Result<Self::Response, Self::Error>>;

    fn poll_ready(&mut self, cx: &mut Context<'_>) -> Poll<Result<(), Self::Error>>;
    fn call(&mut self, req: Request) -> Self::Future;
}
```

Der `Service`-Trait ist für die Verarbeitung von Anfragen und die Rückgabe von Antworten verantwortlich, ähnlich wie Salvos Handler.

### Layer-Trait

```rust
pub trait Layer<S> {
    type Service;
    fn layer(&self, inner: S) -> Self::Service;
}
```

Der `Layer`-Trait wird verwendet, um Dienste zu umhüllen und zusätzliche Funktionalität hinzuzufügen, ähnlich wie Salvos Middleware.

## Salvo-Kompatibilitätsunterstützung

Salvo bietet zwei Schlüsseleigenschaften, um Kompatibilität mit dem Tower-Ökosystem zu erreichen:

- **TowerLayerCompat**: Konvertiert Towers `Layer` in Salvos `Handler`, sodass es als Hoop (Middleware) verwendet werden kann.
- **TowerServiceCompat**: Konvertiert Towers `Service` in Salvos `Handler`.

Anwendungsbeispiel:
```rust
// Verwende Towers RateLimitLayer und konvertiere es über die compat()-Methode in ein Salvo-kompatibles Format
let limit = RateLimitLayer::new(5, Duration::from_secs(30)).compat();
let router = Router::new().hoop(limit).get(hello);
```

# Template-Engines verwenden

import { Tab, Tabs } from '@rspress/core/theme';

Salvo enthält keine integrierte Template-Engine, da die Vorlieben für Template-Engines individuell unterschiedlich sind.

Im Wesentlichen funktioniert eine Template-Engine nach dem Prinzip: Daten + Vorlage = Zeichenkette.

Daher kann jede beliebige Template-Engine unterstützt werden, solange sie letztendlich eine Zeichenkette rendern kann.

Beispielsweise die Unterstützung für `askama`:

  </Tab>
  <Tab label="Cargo.toml">


  <Tab label="templates/hello.html">
  ```html file="<root>/codes/template-askama/templates/hello.html"
  ```
  </Tab>
</Tabs>

**Hinweis**: Für nicht besonders komplexe Projekte empfehlen wir einen Entwicklungsansatz mit Trennung von Frontend und Backend. Dabei werden flexiblere UI-Frameworks mit besserer Ökosystemunterstützung (wie React, Vue, Svelte etc.) für das Frontend verwendet, während Salvo als Backend-API-Dienst dient. Dieser Ansatz ermöglicht eine höhere Entwicklungseffizienz, klarere Verantwortungsbereiche und entspricht besser den modernen Trends in der Webentwicklung.

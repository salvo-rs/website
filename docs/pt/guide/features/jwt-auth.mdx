# Autenticação JWT

JWT (JSON Web Token) é um padrão aberto (RFC 7519) utilizado para transmitir informações de forma segura entre partes. É um método compacto e seguro para URLs que representa declarações na forma de objetos JSON, normalmente usados para autenticação e troca de informações.

O JWT é composto por três partes:
- **Cabeçalho (Header)** - Especifica o tipo do token e o algoritmo de assinatura utilizado
- **Carga útil (Payload)** - Contém as declarações (claims), como ID do usuário, funções, tempo de expiração, etc.
- **Assinatura (Signature)** - Utilizada para verificar se a mensagem não foi alterada durante a transmissão

Uso típico do JWT no fluxo de autenticação:
1. Após o login do usuário, o servidor gera um token JWT
2. O token é retornado ao cliente e armazenado (geralmente em localStorage ou cookie)
3. Nas requisições subsequentes, o cliente inclui este token no cabeçalho Authorization
4. O servidor valida a autenticidade do token e autoriza o acesso

# Authentification de base

Middleware fournissant la prise en charge de l'authentification de base.

## Introduction à l'authentification de base

L'authentification de base est un mécanisme simple d'authentification HTTP qui transmet le nom d'utilisateur et le mot de passe via le champ `Authorization` dans l'en-tête de la requête HTTP. Le format est `Authorization: Basic <base64(username:password)>`. Bien que simple, comme les informations d'identification sont uniquement encodées en Base64 et non chiffrées, elle doit généralement être utilisée avec HTTPS pour garantir la sécurité.

## Comparaison des implémentations de l'authentification de base dans différents frameworks

| Framework | Langage | Implémentation de l'authentification de base |
|-----------|---------|----------------------------------------------|
| Salvo | Rust | Via le middleware `BasicAuth` et une implémentation personnalisée de `BasicAuthValidator` |
| Express | JavaScript | Utilisation du paquet middleware `express-basic-auth` |
| Spring Security | Java | Configuration via `httpBasic()` et implémentation de `UserDetailsService` |
| ASP.NET Core | C# | Utilisation de `.UseAuthentication()` et `AddAuthentication(AuthenticationSchemes.Basic)` |
| Gin | Go | Via un middleware personnalisé ou l'utilisation du paquet `gin-contrib/auth` |

## Cas d'utilisation

L'authentification de base convient aux scénarios suivants :

1. **API et outils internes** : Outils de gestion et API utilisés en interne par l'entreprise
2. **Environnements de développement et de test** : Mise en œuvre rapide de l'authentification sans système de connexion complexe
3. **Protection simple d'API** : Lorsqu'un système complexe de gestion des utilisateurs n'est pas nécessaire
4. **Combinaison avec d'autres mesures de sécurité** : Faisant partie d'une architecture de sécurité à plusieurs couches

Dans Salvo, le middleware d'authentification de base peut être facilement intégré aux routes. Il est très flexible grâce à l'implémentation du trait `BasicAuthValidator` pour personnaliser la logique de validation.

## Points à noter

- Toujours utiliser avec HTTPS pour protéger la transmission des informations d'identification
- Non adapté aux environnements de production stockant des informations sensibles
- Envisager l'utilisation de méthodes d'authentification plus sécurisées comme JWT, OAuth, etc., pour les environnements de production

# Autenticazione di Base

Un middleware che fornisce supporto per l'autenticazione di base.

## Introduzione all'Autenticazione di Base

L'autenticazione di base è un semplice meccanismo di autenticazione HTTP che trasmette nome utente e password tramite l'intestazione `Authorization` nelle richieste HTTP. Il formato è `Authorization: Basic <base64(nomeutente:password)`. Sebbene semplice, poiché le credenziali sono solo codificate in Base64 anziché crittografate, viene tipicamente utilizzata con HTTPS per garantire la sicurezza.

| Framework | Linguaggio | Implementazione Autenticazione di Base |

| Spring Security | Java | Configurata tramite `httpBasic()` e implementata con `UserDetailsService` |
| ASP.NET Core | C# | Utilizza `.UseAuthentication()` e `AddAuthentication(AuthenticationSchemes.Basic)` |
| Gin | Go | Implementata tramite middleware personalizzato o utilizzando il pacchetto `gin-contrib/auth` |

## Casi d'Uso

L'autenticazione di base è adatta per i seguenti scenari:

1. **API e Strumenti Interni**: Strumenti di gestione e API utilizzati all'interno di un'azienda
2. **Ambienti di Sviluppo e Testing**: Implementare rapidamente l'autenticazione senza un sistema di login complesso
3. **Protezione di API Semplici**: Quando non è richiesto un sistema complesso di gestione utenti
4. **Combinata con Altre Misure di Sicurezza**: Come parte di un'architettura di sicurezza multilivello

In Salvo, il middleware di autenticazione di base può essere facilmente integrato nelle rotte. Implementando il tratto `BasicAuthValidator`, è possibile definire logiche di validazione personalizzate, offrendo grande flessibilità.

## Considerazioni

- Utilizzare sempre con HTTPS per proteggere la trasmissione delle credenziali
- Non adatto per ambienti di produzione che memorizzano informazioni sensibili
- Considerare l'uso di metodi di autenticazione più sicuri come JWT o OAuth per ambienti di produzione

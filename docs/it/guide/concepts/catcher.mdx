# Catcher

import { Tab, Tabs } from '@rspress/core/theme';

Quando una `Response` restituisce un codice di stato di errore e il `Body` all'interno della pagina è vuoto, Salvo tenterà di intercettare questo errore utilizzando un `Catcher` e visualizzerà una pagina di errore user-friendly.

Puoi ottenere un `Catcher` predefinito di sistema chiamando `Catcher::default()`, e quindi aggiungerlo al `Service`.

```rust
use salvo::catcher::Catcher;

Service::new(router).catcher(Catcher::default());
```

Il `Catcher` predefinito supporta l'invio di pagine di errore nei formati `XML`, `JSON`, `HTML` e `Text`.

Puoi aggiungere gestori di intercettazione errori personalizzati al `Catcher` collegando `hoop` a questo `Catcher` predefinito. Questi gestori di intercettazione errori sono ancora di tipo `Handler`.

Puoi aggiungere più gestori di intercettazione errori personalizzati al `Catcher` tramite `hoop`. I gestori di errori personalizzati possono chiamare il metodo `FlowCtrl::skip_next` dopo aver elaborato un errore per saltare i gestori di errori successivi e restituire anticipatamente.

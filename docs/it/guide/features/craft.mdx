# Caratteristica Craft

Craft consente agli sviluppatori di generare automaticamente funzioni di gestione ed endpoint tramite semplici annotazioni, integrandosi perfettamente con la generazione di documentazione OpenAPI.

## Casi d'uso

La caratteristica Craft è particolarmente utile nei seguenti scenari:

- Quando è necessario creare rapidamente funzioni di gestione delle route dai metodi di una struttura
- Quando si desidera ridurre il codice boilerplate per l'estrazione manuale dei parametri e la gestione degli errori
- Quando è necessario generare automaticamente documentazione OpenAPI per le proprie API
- Quando si desidera disaccoppiare la logica di business dal framework web

## Utilizzo di base

Per utilizzare la caratteristica Craft è necessario importare i seguenti moduli:

```rust
use salvo::oapi::extract::*;
use salvo::prelude::*;
```

### Creare una struttura di servizio

Utilizzando la macro `#[craft]` per annotare il blocco impl, è possibile convertire i metodi della struttura in funzioni di gestione o endpoint.

```rust
#[derive(Clone)]
pub struct Opts {
    state: i64,
}

#[craft]
impl Opts {
    // Costruttore
    fn new(state: i64) -> Self {
        Self { state }
    }
    
    // Altri metodi...
}
```

### Creare funzioni di gestione

Utilizzare `#[craft(handler)]` per convertire un metodo in una funzione di gestione:

```rust
#[craft(handler)]
fn add1(&self, left: QueryParam<i64>, right: QueryParam<i64>) -> String {
    (self.state + *left + *right).to_string()
}
```

Questo metodo diventerà una funzione di gestione che:
- Estrae automaticamente i valori `left` e `right` dai parametri di query
- Accede allo stato `state` nella struttura
- Restituisce il risultato del calcolo come risposta in formato stringa

### Creare endpoint

Utilizzare `#[craft(endpoint)]` per convertire un metodo in un endpoint:

```rust
#[craft(endpoint)]
pub(crate) fn add2(
    self: ::std::sync::Arc<Self>,
    left: QueryParam<i64>,
    right: QueryParam<i64>,
) -> String {
    (self.state + *left + *right).to_string()
}
```

Gli endpoint possono utilizzare `Arc` per condividere lo stato, particolarmente utile quando si gestiscono richieste concorrenti.

### Endpoint statici

È possibile creare anche endpoint statici che non dipendono dallo stato dell'istanza:

```rust
#[craft(endpoint(responses((status_code = 400, description = "Wrong request parameters."))))]
pub fn add3(left: QueryParam<i64>, right: QueryParam<i64>) -> String {
    (*left + *right).to_string()
}
```

In questo esempio, è stata aggiunta anche una descrizione personalizzata per la risposta di errore, che si rifletterà nella documentazione OpenAPI generata.

## Estrattori di parametri

Il modulo `oapi::extract` di Salvo fornisce vari estrattori di parametri, i più comuni includono:

- `QueryParam<T>`: estrae parametri dalla stringa di query
- `PathParam<T>`: estrae parametri dal percorso URL
- `FormData<T>`: estrae parametri dai dati del form
- `JsonBody<T>`: estrae parametri dal corpo della richiesta JSON

Questi estrattori eseguono automaticamente l'analisi dei parametri e la conversione dei tipi, semplificando notevolmente la scrittura delle funzioni di gestione.

## Integrazione con OpenAPI

La caratteristica Craft può generare automaticamente documentazione API conforme alle specifiche OpenAPI. Nell'esempio:

```rust
let router = Router::new()
    .push(Router::with_path("add1").get(opts.add1()))
    .push(Router::with_path("add2").get(opts.add2()))
    .push(Router::with_path("add3").get(Opts::add3()));

// Generare documentazione OpenAPI
let doc = OpenApi::new("Example API", "0.0.1").merge_router(&router);

// Aggiungere rotte per la documentazione OpenAPI e Swagger UI
let router = router
    .push(doc.into_router("/api-doc/openapi.json"))
    .push(SwaggerUi::new("/api-doc/openapi.json").into_router("swagger-ui"));
```

Dopo questa configurazione, la documentazione API sarà disponibile all'endpoint `/api-doc/openapi.json`, e Swagger UI sarà accessibile al percorso `/swagger-ui`.

## Esempio completo

Di seguito è riportato un esempio completo che mostra come utilizzare la caratteristica Craft per creare tre tipi diversi di endpoint:

```rust
use salvo::oapi::extract::*;
use salvo::prelude::*;
use std::sync::Arc;

#[derive(Clone)]
pub struct Opts {
    state: i64,
}

#[craft]
impl Opts {
    fn new(state: i64) -> Self {
        Self { state }
    }

    #[craft(handler)]
    fn add1(&self, left: QueryParam<i64>, right: QueryParam<i64>) -> String {
        (self.state + *left + *right).to_string()
    }

    #[craft(endpoint)]
    pub(crate) fn add2(
        self: ::std::sync::Arc<Self>,
        left: QueryParam<i64>,
        right: QueryParam<i64>,
    ) -> String {
        (self.state + *left + *right).to_string()
    }

    #[craft(endpoint(responses((status_code = 400, description = "Wrong request parameters."))))]
    pub fn add3(left: QueryParam<i64>, right: QueryParam<i64>) -> String {
        (*left + *right).to_string()
    }
}

#[tokio::main]
async fn main() {
    // Creare uno stato condiviso, valore iniziale 1
    let opts = Arc::new(Opts::new(1));

    // Configurare le rotte per i tre endpoint
    let router = Router::new()
        .push(Router::with_path("add1").get(opts.add1()))
        .push(Router::with_path("add2").get(opts.add2()))
        .push(Router::with_path("add3").get(Opts::add3()));

    // Generare documentazione OpenAPI
    let doc = OpenApi::new("Example API", "0.0.1").merge_router(&router);
    
    // Aggiungere rotte per la documentazione OpenAPI e Swagger UI
    let router = router
        .push(doc.into_router("/api-doc/openapi.json"))
        .push(SwaggerUi::new("/api-doc/openapi.json").into_router("swagger-ui"));

    // Avviare il server su localhost:5800
    let acceptor = TcpListener::new("127.0.0.1:5800").bind().await;
    Server::new(acceptor).serve(router).await;
}
```

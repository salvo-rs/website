# レスポンスの圧縮

`Response` の内容を圧縮処理するミドルウェアです。

以下の3つの圧縮形式に対応しています: `br`, `gzip`, `deflate`。各圧縮方式の優先度などを設定可能です。

## HTTPプロトコルにおけるデータ圧縮

1. ファイル形式レベル：特定のファイルタイプに特化した圧縮アルゴリズムを使用
2. HTTPプロトコルレベル：コンテンツエンコーディングによるエンドツーエンドのリソース圧縮転送
3. コネクションレベル：HTTP接続のノード間でデータ圧縮を実施

HTTP圧縮に関する詳細は [Mozilla Developer Networkのドキュメント](https://developer.mozilla.org/ja/docs/Web/HTTP/Compression) を参照してください。

# Writer

O `Writer` é utilizado para escrever conteúdo na `Response`:

```rust
#[async_trait]
pub trait Writer {
    async fn write(mut self, req: &mut Request, depot: &mut Depot, res: &mut Response);
}
```

Em comparação com o Handler:

```rust
#[async_trait]
pub trait Handler: Send + Sync + 'static {
    async fn handle(&self, req: &mut Request, depot: &mut Depot, res: &mut Response, ctrl: &mut FlowCtrl);
}
```

As diferenças diretas entre eles são:
- A finalidade é diferente. O `Writer` representa a escrita de conteúdo específico na `Response`, implementado por conteúdos concretos, como strings, mensagens de erro, etc. Já o `Handler` é utilizado para processar toda a requisição.
- O `Writer` é criado dentro do `Handler` e é consumido quando a função `write` é chamada, sendo uma chamada única. Enquanto isso, o `Handler` é compartilhado por todas as requisições.
- O `Writer` pode ser retornado como conteúdo no `Result` do `Handler`.
- O `Writer` não possui o parâmetro `FlowCtrl`, portanto não pode controlar o fluxo de execução da requisição.

O `Scribe` implementa o `Writer`, mas com funcionalidades mais limitadas:

```rust
pub trait Scribe {
    fn render(self, res: &mut Response);
}
```

A função de renderização do `Scribe` apenas escreve dados na `Response`, e esse processo não pode obter informações do `Request` ou do `Depot`.
